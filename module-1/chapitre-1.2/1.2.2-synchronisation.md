## Les primitives de synchronisation Tokio

Tokio fournit des équivalents async de toutes les primitives de synchronisation standard. Contrairement à `std::sync`, ces versions libèrent le thread pendant l'attente plutôt que de le bloquer.

---

## Mutex\<T\> async — Usage correct

```rust
use tokio::sync::Mutex;
use std::sync::Arc;

// ❌ Anti-pattern : tenir un verrou async pendant une opération I/O
async fn mauvais_usage(mutex: Arc<Mutex<Vec<String>>>) {
    let mut garde = mutex.lock().await;
    
    // Le verrou est tenu pendant toute la requête HTTP !
    // Toutes les autres tâches attendant ce mutex sont bloquées
    let données = reqwest::get("https://api.example.com/data")
        .await.unwrap()
        .text().await.unwrap();
    
    garde.push(données);
} // Verrou libéré seulement ici

// ✅ Pattern correct : minimiser le temps de verrouillage
async fn bon_usage(mutex: Arc<Mutex<Vec<String>>>) {
    // 1. Faire l'opération I/O SANS le verrou
    let données = reqwest::get("https://api.example.com/data")
        .await.unwrap()
        .text().await.unwrap();
    
    // 2. Tenir le verrou seulement pour l'écriture (microseconds)
    {
        let mut garde = mutex.lock().await;
        garde.push(données);
    } // Verrou libéré immédiatement
}
```

## RwLock\<T\> — Lectures parallèles

```rust
use tokio::sync::RwLock;
use std::sync::Arc;
use std::collections::HashMap;

type RegistreServices = Arc<RwLock<HashMap<String, String>>>;

async fn enregistrer_service(
    registre: &RegistreServices,
    nom: &str,
    adresse: &str,
) {
    let mut écriture = registre.write().await; // Verrou exclusif
    écriture.insert(nom.to_string(), adresse.to_string());
    tracing::info!("Service '{}' enregistré sur {}", nom, adresse);
}

async fn découvrir_service(
    registre: &RegistreServices,
    nom: &str,
) -> Option<String> {
    let lecture = registre.read().await; // Verrou partagé (N lecteurs simultanés)
    lecture.get(nom).cloned()
}

#[tokio::main]
async fn main() {
    let registre: RegistreServices = Arc::new(RwLock::new(HashMap::new()));
    
    // Une écriture
    enregistrer_service(&registre, "auth-service", "10.0.0.1:8080").await;
    enregistrer_service(&registre, "user-service", "10.0.0.2:8080").await;
    
    // Plusieurs lectures simultanées — aucune contention entre elles
    let r1 = Arc::clone(&registre);
    let r2 = Arc::clone(&registre);
    let r3 = Arc::clone(&registre);
    
    tokio::join!(
        async move { println!("{:?}", découvrir_service(&r1, "auth-service").await) },
        async move { println!("{:?}", découvrir_service(&r2, "user-service").await) },
        async move { println!("{:?}", découvrir_service(&r3, "auth-service").await) },
    );
}
```

## Semaphore — Contrôle précis de la concurrence

```rust
use tokio::sync::Semaphore;
use std::sync::Arc;

// Pattern : pool de connexions simplifié
struct PoolConnexions {
    sémaphore: Arc<Semaphore>,
    max: usize,
}

impl PoolConnexions {
    fn nouveau(max: usize) -> Self {
        PoolConnexions {
            sémaphore: Arc::new(Semaphore::new(max)),
            max,
        }
    }
    
    async fn acquérir(&self) -> ConnexionGuard {
        let permis = self.sémaphore.clone().acquire_owned().await.unwrap();
        ConnexionGuard { _permis: permis }
    }
    
    fn disponibles(&self) -> usize {
        self.sémaphore.available_permits()
    }
}

struct ConnexionGuard {
    _permis: tokio::sync::OwnedSemaphorePermit,
    // Connexion libérée quand OwnedSemaphorePermit est droppé
}

// Limiter les accès concurrents à une ressource externe (API tierce, DB)
async fn appel_api_limitée(
    pool: Arc<PoolConnexions>,
    url: &str,
) -> reqwest::Result<String> {
    let _guard = pool.acquérir().await;
    println!("Connexions disponibles : {}", pool.disponibles());
    
    // Exactement `max` appels simultanés maximum
    reqwest::get(url).await?.text().await
}
```

## Barrier — Synchroniser plusieurs tâches à un point

```rust
use tokio::sync::Barrier;
use std::sync::Arc;

#[tokio::main]
async fn main() {
    let nb_workers = 5;
    let barrière = Arc::new(Barrier::new(nb_workers));
    
    let mut handles = vec![];
    
    for i in 0..nb_workers {
        let b = Arc::clone(&barrière);
        
        handles.push(tokio::spawn(async move {
            // Phase 1 : initialisation
            println!("Worker {} : initialisation...", i);
            tokio::time::sleep(
                std::time::Duration::from_millis(i as u64 * 100)
            ).await;
            
            // Attendre que TOUS les workers aient fini l'initialisation
            let résultat = b.wait().await;
            
            if résultat.is_leader() {
                println!("=== Tous les workers sont prêts — démarrage ! ===");
            }
            
            // Phase 2 : traitement (commence simultanément pour tous)
            println!("Worker {} : traitement démarré", i);
        }));
    }
    
    for h in handles { h.await.unwrap(); }
}
```

## 🔥 Failure First — Détecter et résoudre les deadlocks async

### Technique 1 : Timeout sur l'acquisition du verrou

```rust
use tokio::time::{timeout, Duration};
use tokio::sync::Mutex;

async fn acquérir_avec_timeout<T>(
    mutex: &Mutex<T>,
    timeout_dur: Duration,
) -> Option<tokio::sync::MutexGuard<'_, T>> {
    match timeout(timeout_dur, mutex.lock()).await {
        Ok(garde) => Some(garde),
        Err(_) => {
            // Timeout ! Probablement un deadlock
            tracing::error!(
                "Timeout d'acquisition du mutex après {:?} — deadlock possible ?",
                timeout_dur
            );
            None
        }
    }
}
```

### Technique 2 : try_lock pour diagnostiquer

```rust
use tokio::sync::Mutex;

async fn diagnostiquer_contention(mutex: &Mutex<Vec<i32>>) {
    match mutex.try_lock() {
        Ok(garde) => {
            println!("Verrou acquis immédiatement — pas de contention");
            drop(garde);
        }
        Err(_) => {
            println!("Verrou occupé — contention détectée");
            // Attente normale
            let _garde = mutex.lock().await;
        }
    }
}
```

## Atomic — Pour les compteurs sans verrou

```rust
use std::sync::atomic::{AtomicU64, AtomicBool, Ordering};
use std::sync::Arc;

struct MétriqueServeur {
    requêtes_totales: AtomicU64,
    requêtes_actives: AtomicU64,
    en_arrêt: AtomicBool,
}

impl MétriqueServeur {
    fn nouvelle() -> Arc<Self> {
        Arc::new(MétriqueServeur {
            requêtes_totales: AtomicU64::new(0),
            requêtes_actives: AtomicU64::new(0),
            en_arrêt: AtomicBool::new(false),
        })
    }
    
    fn début_requête(&self) {
        self.requêtes_totales.fetch_add(1, Ordering::Relaxed);
        self.requêtes_actives.fetch_add(1, Ordering::Relaxed);
    }
    
    fn fin_requête(&self) {
        self.requêtes_actives.fetch_sub(1, Ordering::Relaxed);
    }
    
    fn déclencher_arrêt(&self) {
        self.en_arrêt.store(true, Ordering::SeqCst);
    }
    
    fn afficher(&self) {
        println!(
            "Total: {}, Actives: {}, Arrêt: {}",
            self.requêtes_totales.load(Ordering::Relaxed),
            self.requêtes_actives.load(Ordering::Relaxed),
            self.en_arrêt.load(Ordering::Relaxed),
        );
    }
}
```

## Choisir la bonne primitive

```
Accès concurrent en lecture/écriture à des données simples
├─ Compteur, flag → Atomic (pas de verrou du tout)
├─ Données complexes, beaucoup de lectures → RwLock
└─ Données complexes, beaucoup d'écritures → Mutex

Limiter la concurrence
└─ Semaphore

Communication entre tâches
├─ File de travail → mpsc
├─ Notifications à tous → broadcast
├─ Requête/réponse → oneshot
└─ État observable → watch

Synchronisation d'étapes
└─ Barrier

Architecture sans partage d'état
└─ Actor model (mpsc + oneshot)
```
