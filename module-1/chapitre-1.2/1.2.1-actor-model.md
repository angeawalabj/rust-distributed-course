# 1.2.1 — Actor Model : Concevoir des systèmes concurrents sans partage d'état

> **Prérequis :** [1.1.3 — Gestion des tâches](../chapitre-1.1/1.1.3-gestion-taches.md)
> **Durée estimée :** 65 minutes

---

## Le problème du partage d'état

La concurrence par partage d'état — plusieurs threads ou tâches qui accèdent aux mêmes données via des Mutex — est fonctionnelle mais fragile. Elle requiert une discipline constante sur l'ordre des verrous, les sections critiques et la gestion des pannes.

Le **modèle acteur** propose une alternative : **chaque acteur possède ses propres données, qui ne sont jamais partagées**. La communication se fait exclusivement par échange de messages. C'est l'approche d'Erlang, de Akka (Scala), et c'est ce que Tokio permet d'implémenter naturellement.

---

## Anatomie d'un acteur

Un acteur en Rust/Tokio, c'est :
1. **Une tâche Tokio** — la boucle de traitement
2. **Une boîte de réception** — le channel `mpsc`
3. **Un état privé** — invisible depuis l'extérieur
4. **Un handle** — l'interface publique pour envoyer des messages

```
┌─────────────────────────────────────┐
│             Acteur                  │
│  ┌─────────────────────────────┐    │
│  │  État privé (non partagé)   │    │
│  │  - compteur: i64            │    │
│  │  - config: Config           │    │
│  │  - connexion: DbPool        │    │
│  └─────────────────────────────┘    │
│  ┌─────────────────────────────┐    │
│  │  Boucle de traitement       │◄───┼─── Messages entrants
│  │  while let Some(msg) = rx.. │    │
│  └─────────────────────────────┘    │
└─────────────────────────────────────┘
         ▲
         │ Handle (clone possible)
         │
┌────────┴──────────────────────┐
│  Code appelant (autre acteur) │
│  handle.envoyer(message).await│
└───────────────────────────────┘
```

## Implémentation complète d'un acteur

Voici un acteur de cache avec TTL (Time-To-Live) :

```rust
use std::collections::HashMap;
use std::time::{Duration, Instant};
use tokio::sync::{mpsc, oneshot};

// ── Types de messages ──────────────────────────────────────────────────

#[derive(Debug)]
enum MessageCache {
    Obtenir {
        clé: String,
        réponse: oneshot::Sender<Option<String>>,
    },
    Définir {
        clé: String,
        valeur: String,
        ttl: Duration,
    },
    Supprimer {
        clé: String,
    },
    Vider,
    Statistiques {
        réponse: oneshot::Sender<StatsCache>,
    },
}

#[derive(Debug, Clone)]
pub struct StatsCache {
    pub entrées: usize,
    pub succès: u64,
    pub échecs: u64,
}

// ── Entrée de cache ────────────────────────────────────────────────────

struct EntréeCache {
    valeur: String,
    expiration: Instant,
}

impl EntréeCache {
    fn est_expirée(&self) -> bool {
        Instant::now() > self.expiration
    }
}

// ── L'acteur lui-même ──────────────────────────────────────────────────

struct ActeurCache {
    données: HashMap<String, EntréeCache>,
    stats: StatsCache,
}

impl ActeurCache {
    fn nouveau() -> Self {
        ActeurCache {
            données: HashMap::new(),
            stats: StatsCache { entrées: 0, succès: 0, échecs: 0 },
        }
    }
    
    fn traiter(&mut self, msg: MessageCache) {
        // Nettoyage des entrées expirées à chaque opération (lazy cleanup)
        self.nettoyer_expirés();
        
        match msg {
            MessageCache::Obtenir { clé, réponse } => {
                let résultat = self.données
                    .get(&clé)
                    .filter(|e| !e.est_expirée())
                    .map(|e| e.valeur.clone());
                
                if résultat.is_some() {
                    self.stats.succès += 1;
                } else {
                    self.stats.échecs += 1;
                }
                
                let _ = réponse.send(résultat);
            }
            
            MessageCache::Définir { clé, valeur, ttl } => {
                self.données.insert(clé, EntréeCache {
                    valeur,
                    expiration: Instant::now() + ttl,
                });
                self.stats.entrées = self.données.len();
            }
            
            MessageCache::Supprimer { clé } => {
                self.données.remove(&clé);
                self.stats.entrées = self.données.len();
            }
            
            MessageCache::Vider => {
                self.données.clear();
                self.stats.entrées = 0;
            }
            
            MessageCache::Statistiques { réponse } => {
                let _ = réponse.send(self.stats.clone());
            }
        }
    }
    
    fn nettoyer_expirés(&mut self) {
        self.données.retain(|_, e| !e.est_expirée());
        self.stats.entrées = self.données.len();
    }
    
    async fn run(mut self, mut rx: mpsc::Receiver<MessageCache>) {
        // Nettoyage périodique toutes les 60 secondes
        let mut intervalle_nettoyage = tokio::time::interval(Duration::from_secs(60));
        
        loop {
            tokio::select! {
                msg = rx.recv() => {
                    match msg {
                        Some(message) => self.traiter(message),
                        None => {
                            // Tous les handles sont droppés → arrêt
                            tracing::info!("Cache acteur : arrêt (plus de senders)");
                            break;
                        }
                    }
                }
                _ = intervalle_nettoyage.tick() => {
                    let avant = self.données.len();
                    self.nettoyer_expirés();
                    let supprimés = avant - self.données.len();
                    if supprimés > 0 {
                        tracing::debug!("{} entrées expirées supprimées", supprimés);
                    }
                }
            }
        }
    }
}

// ── Le Handle — interface publique ─────────────────────────────────────

#[derive(Clone)]
pub struct HandleCache {
    tx: mpsc::Sender<MessageCache>,
}

impl HandleCache {
    pub fn démarrer() -> Self {
        let (tx, rx) = mpsc::channel(256);
        tokio::spawn(ActeurCache::nouveau().run(rx));
        HandleCache { tx }
    }
    
    pub async fn obtenir(&self, clé: &str) -> Option<String> {
        let (réponse_tx, réponse_rx) = oneshot::channel();
        self.tx.send(MessageCache::Obtenir {
            clé: clé.to_string(),
            réponse: réponse_tx,
        }).await.ok()?;
        réponse_rx.await.ok()?
    }
    
    pub async fn définir(&self, clé: &str, valeur: &str, ttl: Duration) {
        let _ = self.tx.send(MessageCache::Définir {
            clé: clé.to_string(),
            valeur: valeur.to_string(),
            ttl,
        }).await;
    }
    
    pub async fn statistiques(&self) -> Option<StatsCache> {
        let (tx, rx) = oneshot::channel();
        self.tx.send(MessageCache::Statistiques { réponse: tx }).await.ok()?;
        rx.await.ok()
    }
}

// ── Utilisation ────────────────────────────────────────────────────────

#[tokio::main]
async fn main() {
    let cache = HandleCache::démarrer();
    
    // Utilisable depuis plusieurs tâches simultanément (Clone)
    let cache2 = cache.clone();
    
    // Écrire
    cache.définir("utilisateur:42", "Alice", Duration::from_secs(300)).await;
    
    // Lire depuis une autre tâche
    tokio::spawn(async move {
        if let Some(val) = cache2.obtenir("utilisateur:42").await {
            println!("Trouvé : {}", val);
        }
    }).await.unwrap();
    
    // Statistiques
    if let Some(stats) = cache.statistiques().await {
        println!("Succès: {}, Échecs: {}", stats.succès, stats.échecs);
    }
}
```

## Supervision d'acteurs — Redémarrer en cas de panique

```rust
use tokio::task::JoinHandle;

async fn superviser_acteur<F, Fut>(
    nom: &'static str,
    créer: F,
    max_redémarrages: u32,
)
where
    F: Fn() -> Fut + Send + 'static,
    Fut: std::future::Future<Output = ()> + Send + 'static,
{
    let mut redémarrages = 0u32;
    
    loop {
        let handle: JoinHandle<()> = tokio::spawn(créer());
        
        match handle.await {
            Ok(()) => {
                tracing::info!("{} s'est arrêté normalement", nom);
                break;
            }
            Err(e) if e.is_panic() => {
                redémarrages += 1;
                if redémarrages > max_redémarrages {
                    tracing::error!("{} a dépassé le nombre max de redémarrages", nom);
                    break;
                }
                
                let délai = std::time::Duration::from_millis(
                    100 * 2u64.pow(redémarrages.min(10))
                );
                tracing::warn!("{} a paniqué (redémarrage {}/{}) dans {:?}",
                    nom, redémarrages, max_redémarrages, délai);
                
                tokio::time::sleep(délai).await;
            }
            Err(_) => break,
        }
    }
}
```

## 🔥 Failure First — Acteur avec boîte de réception pleine

```rust
#[tokio::main]
async fn main() {
    // Buffer très petit pour reproduire le problème
    let (tx, mut rx) = mpsc::channel::<String>(2);
    
    // Consumer lent
    tokio::spawn(async move {
        while let Some(msg) = rx.recv().await {
            tokio::time::sleep(std::time::Duration::from_secs(1)).await;
            println!("Traité : {}", msg);
        }
    });
    
    // Producteur rapide — va remplir le buffer
    for i in 0..10 {
        match tx.try_send(format!("Message {}", i)) {
            Ok(()) => println!("Envoyé {}", i),
            Err(mpsc::error::TrySendError::Full(msg)) => {
                eprintln!("⚠️  Buffer plein ! Message {} perdu ou à retry", msg);
            }
            Err(mpsc::error::TrySendError::Closed(_)) => break,
        }
    }
}
```

**Solutions selon le contexte :**
- Augmenter la taille du buffer (bande passante accrue, mais mémoire)
- Utiliser `send().await` (backpressure — le producteur attend)
- Utiliser `try_send()` avec logique de retry et circuit breaker
- Réduire le temps de traitement du consumer

## Résumé — Quand utiliser le modèle acteur

| Situation | Acteur | Mutex partagé |
|---|---|---|
| État complexe avec beaucoup de lectures | ✅ | ✅ (RwLock) |
| État modifié fréquemment | ✅ | ⚠️ Contention |
| Logique de nettoyage périodique | ✅ (select! + interval) | ⚠️ Complexe |
| État partagé très simple (compteur) | ⚠️ Overhead | ✅ (Atomic) |
| Supervision et restart | ✅ | ❌ |
| Debugging de concurrence | ✅ (état isolé) | ⚠️ Difficile |
