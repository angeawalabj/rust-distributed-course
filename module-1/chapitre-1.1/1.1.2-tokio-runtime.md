## L'architecture interne de Tokio

Tokio n'est pas magique — c'est un orchestrateur de tâches. Comprendre son fonctionnement vous permet de l'utiliser efficacement et de diagnostiquer les problèmes de performance.

```
Architecture Tokio (multi-thread scheduler) :

Thread 1           Thread 2           Thread 3           Thread 4
┌──────────┐       ┌──────────┐       ┌──────────┐       ┌──────────┐
│ File de  │       │ File de  │       │ File de  │       │ File de  │
│ tâches   │       │ tâches   │       │ tâches   │       │ tâches   │
│ locale   │       │ locale   │       │ locale   │       │ locale   │
└────┬─────┘       └────┬─────┘       └────┬─────┘       └────┬─────┘
     │                  │                  │                  │
     └──────────────────┴──────────────────┴──────────────────┘
                              │
                    File globale de tâches
                              │
                    I/O Driver (epoll/kqueue)
```

Quand une tâche est bloquée sur de l'I/O, elle est suspendue et le thread passe à la prochaine tâche disponible. Quand l'I/O est prête, la tâche est replacée dans la file.

## Configurer le runtime Tokio

```rust
use tokio::runtime;

fn main() {
    // Configuration fine du runtime
    let rt = runtime::Builder::new_multi_thread()
        .worker_threads(4)              // Nombre de threads worker (défaut: nCPU)
        .max_blocking_threads(512)      // Pool pour spawn_blocking
        .thread_stack_size(2 * 1024 * 1024) // Stack par thread (2 Mo)
        .thread_name("mon-service-worker") // Nom visible dans les outils de profiling
        .enable_all()                   // Active I/O et timers
        .build()
        .expect("Impossible de créer le runtime Tokio");
    
    rt.block_on(async {
        // Code principal ici
        run_serveur().await;
    });
}

// Pour les tests : runtime single-thread (déterministe)
#[cfg(test)]
mod tests {
    #[tokio::test(flavor = "current_thread")]
    async fn test_simple() {
        // S'exécute sur un seul thread — plus prévisible
    }
}
```

## Spawner des tâches concurrentes

```rust
use tokio::task::{self, JoinHandle};
use std::time::Duration;

#[tokio::main]
async fn main() {
    // spawn — tâche qui vit indépendamment (peut outlive l'appelant)
    let handle: JoinHandle<i32> = task::spawn(async {
        tokio::time::sleep(Duration::from_millis(100)).await;
        42
    });
    
    // Attendre la tâche et récupérer son résultat
    match handle.await {
        Ok(val) => println!("Résultat : {}", val),
        Err(e) if e.is_panic() => println!("La tâche a paniqué !"),
        Err(e) if e.is_cancelled() => println!("La tâche a été annulée"),
        Err(e) => println!("Erreur inattendue : {}", e),
    }
    
    // spawn_local — pour les tâches non-Send (thread courant seulement)
    let local = task::LocalSet::new();
    local.run_until(async {
        task::spawn_local(async {
            // Peut utiliser des types Rc<T> non thread-safe
        }).await.unwrap();
    }).await;
}
```

## JoinSet — Gérer un groupe de tâches

```rust
use tokio::task::JoinSet;

async fn traiter_urls(urls: Vec<String>) -> Vec<Result<String, reqwest::Error>> {
    let mut set = JoinSet::new();
    
    // Lancer toutes les requêtes concurremment
    for url in urls {
        set.spawn(async move {
            reqwest::get(&url).await?.text().await
        });
    }
    
    let mut résultats = Vec::new();
    
    // Collecter les résultats dans l'ordre de complétion
    while let Some(résultat) = set.join_next().await {
        match résultat {
            Ok(Ok(corps)) => résultats.push(Ok(corps)),
            Ok(Err(e)) => résultats.push(Err(e)),
            Err(e) => eprintln!("Tâche échouée : {}", e), // Panic dans la tâche
        }
    }
    
    résultats
}
```

## spawn_blocking — Code CPU-intensif ou bloquant

```rust
use tokio::task;

async fn pipeline_traitement(données: Vec<u8>) -> Vec<u8> {
    // Étape 1 : lecture async (I/O)
    let données = tokio::fs::read("fichier.bin").await.unwrap();
    
    // Étape 2 : traitement CPU-intensif (code synchrone)
    // CRUCIAL : ne jamais faire ça directement dans async !
    let compressé = task::spawn_blocking(move || {
        // Ce code s'exécute dans un thread dédié, hors du runtime
        // Peut utiliser std::thread::sleep, CPU-heavy, etc.
        compresser_gzip(données)
    }).await.unwrap();
    
    // Étape 3 : écriture async (I/O)
    tokio::fs::write("sortie.bin.gz", &compressé).await.unwrap();
    
    compressé
}

fn compresser_gzip(données: Vec<u8>) -> Vec<u8> {
    use flate2::write::GzEncoder;
    use flate2::Compression;
    use std::io::Write;
    
    let mut encodeur = GzEncoder::new(Vec::new(), Compression::default());
    encodeur.write_all(&données).unwrap();
    encodeur.finish().unwrap()
}
```

## Backpressure — Ne pas se noyer sous les tâches

Un serveur sans contrôle de flux peut créer des milliers de tâches et s'effondrer sous la pression mémoire. La **backpressure** est le mécanisme qui limite la charge.

```rust
use tokio::sync::Semaphore;
use std::sync::Arc;

#[tokio::main]
async fn main() {
    // Limiter à 100 requêtes concurrentes maximum
    let sémaphore = Arc::new(Semaphore::new(100));
    
    let requêtes: Vec<String> = (0..10_000)
        .map(|i| format!("https://api.example.com/item/{}", i))
        .collect();
    
    let mut tâches = vec![];
    
    for url in requêtes {
        let sem = Arc::clone(&sémaphore);
        
        let tâche = tokio::spawn(async move {
            // Acquiert un permis — se bloque si 100 déjà actives
            let _permis = sem.acquire().await.unwrap();
            // _permis est libéré automatiquement quand il sort du scope
            
            traiter_requête(&url).await
        });
        
        tâches.push(tâche);
    }
    
    // Attendre toutes les tâches
    for tâche in tâches {
        let _ = tâche.await;
    }
}

async fn traiter_requête(url: &str) -> String {
    // Simulation
    tokio::time::sleep(std::time::Duration::from_millis(10)).await;
    format!("Réponse de {}", url)
}
```

## Timeouts et annulation

```rust
use tokio::time::{timeout, Duration};
use tokio_util::sync::CancellationToken;

async fn avec_timeout() {
    // Timeout simple
    let résultat = timeout(
        Duration::from_secs(5),
        opération_lente()
    ).await;
    
    match résultat {
        Ok(val) => println!("OK : {:?}", val),
        Err(_) => println!("Timeout après 5 secondes"),
    }
}

// Annulation propagée — utile pour shutdown graceful
async fn traitement_annulable(token: CancellationToken) {
    loop {
        tokio::select! {
            _ = token.cancelled() => {
                println!("Annulation demandée — arrêt propre");
                break;
            }
            _ = tokio::time::sleep(Duration::from_secs(1)) => {
                println!("Traitement d'un batch...");
            }
        }
    }
}

#[tokio::main]
async fn main() {
    let token = CancellationToken::new();
    let token_clone = token.clone();
    
    let tâche = tokio::spawn(traitement_annulable(token_clone));
    
    // Simuler un signal d'arrêt après 3 secondes
    tokio::time::sleep(Duration::from_secs(3)).await;
    token.cancel(); // Envoyer le signal d'annulation
    
    tâche.await.unwrap();
    println!("Arrêt propre terminé");
}
```

## 🔥 Failure First — Diagnostiquer le "runtime starvation"

```rust
use std::time::Instant;

#[tokio::main]
async fn main() {
    // Lancer un moniteur de latence du runtime
    tokio::spawn(async {
        loop {
            let début = Instant::now();
            tokio::task::yield_now().await; // Cède le thread
            let latence = début.elapsed();
            
            if latence > std::time::Duration::from_millis(10) {
                eprintln!("⚠️  Runtime starvation détecté ! Latence: {:?}", latence);
            }
        }
    });
    
    // Simuler une tâche qui bloque le runtime
    tokio::spawn(async {
        // ❌ MAUVAIS : calcul CPU bloquant dans async
        let mut somme = 0u64;
        for i in 0..1_000_000_000u64 {
            somme += i;
        }
        println!("Somme : {}", somme);
    });
    
    tokio::time::sleep(std::time::Duration::from_secs(5)).await;
}
```

Sortie :
```
⚠️  Runtime starvation détecté ! Latence: 2.3s
⚠️  Runtime starvation détecté ! Latence: 1.8s
Somme : 499999999500000000
```

**Correction :** Utiliser `spawn_blocking` pour le calcul CPU.

## Métriques Tokio — Observer le runtime

```toml
[dependencies]
tokio = { version = "1", features = ["full", "rt", "metrics"] }
```

```rust
#[tokio::main]
async fn main() {
    let métriques = tokio::runtime::Handle::current().metrics();
    
    tokio::spawn(async move {
        loop {
            println!(
                "Workers: {}, Tâches actives: {}, Tâches en file: {}",
                métriques.num_workers(),
                métriques.active_tasks_count(),
                métriques.injection_queue_depth(),
            );
            tokio::time::sleep(std::time::Duration::from_secs(1)).await;
        }
    });
    
    // ... reste de l'application
}
```

## Résumé

| Concept | API | Quand utiliser |
|---|---|---|
| Tâche concurrente | `tokio::spawn` | I/O concurrent |
| Code bloquant | `task::spawn_blocking` | CPU-intensif, std::thread |
| Groupe de tâches | `JoinSet` | Batch de tâches parallèles |
| Limiter la concurrence | `Semaphore` | Backpressure |
| Timeout | `tokio::time::timeout` | Éviter les blocages infinis |
| Annulation | `CancellationToken` | Shutdown graceful |
