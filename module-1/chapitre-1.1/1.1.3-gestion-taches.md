## Channels async — La colonne vertébrale des architectures async

Les channels Tokio permettent la communication entre tâches. Il en existe trois types, chacun conçu pour un besoin précis.

### mpsc — Multi-Producer Single-Consumer

```rust
use tokio::sync::mpsc;

#[tokio::main]
async fn main() {
    // Créer un channel avec buffer de 32 messages
    let (tx, mut rx) = mpsc::channel::<String>(32);
    
    // Plusieurs producteurs (clones du sender)
    for i in 0..5 {
        let tx_clone = tx.clone();
        tokio::spawn(async move {
            for j in 0..3 {
                tx_clone
                    .send(format!("Producteur {} message {}", i, j))
                    .await
                    .expect("Receiver fermé");
            }
        });
    }
    
    // Fermer l'original pour que le receiver sache quand tous ont fini
    drop(tx);
    
    // Un seul consommateur
    while let Some(msg) = rx.recv().await {
        println!("Reçu : {}", msg);
    }
    println!("Tous les producteurs ont terminé");
}
```

**Backpressure automatique :** Quand le buffer est plein, `send().await` se bloque jusqu'à ce qu'il y ait de la place. C'est la backpressure naturelle du canal.

```rust
// send_timeout — pour éviter un blocage indéfini
use std::time::Duration;

match tx.send_timeout(message, Duration::from_secs(5)).await {
    Ok(()) => println!("Envoyé"),
    Err(mpsc::error::SendTimeoutError::Timeout(msg)) => {
        eprintln!("Le consumer est trop lent ! Message abandonné : {}", msg);
    }
    Err(mpsc::error::SendTimeoutError::Closed(msg)) => {
        eprintln!("Le consumer est mort. Message perdu : {}", msg);
    }
}
```

### broadcast — Un-to-Many

```rust
use tokio::sync::broadcast;

#[tokio::main]
async fn main() {
    // Chaque subscriber reçoit TOUS les messages
    let (tx, _) = broadcast::channel::<String>(16);
    
    // Créer des subscribers
    let mut rx1 = tx.subscribe();
    let mut rx2 = tx.subscribe();
    let mut rx3 = tx.subscribe();
    
    // Envoyer
    tx.send("Message pour tous".to_string()).unwrap();
    tx.send("Deuxième message".to_string()).unwrap();
    
    // Chaque receiver a sa propre copie
    println!("rx1: {}", rx1.recv().await.unwrap());
    println!("rx2: {}", rx2.recv().await.unwrap());
    println!("rx3: {}", rx3.recv().await.unwrap());
}
```

> ⚠️ Si un subscriber est trop lent, les anciens messages sont écrasés (`Lagged` error). Ajustez la capacité du channel en conséquence.

### oneshot — Une seule réponse

```rust
use tokio::sync::oneshot;

// Pattern classique : requête/réponse
struct RequêteBDD {
    sql: String,
    réponse: oneshot::Sender<Result<Vec<String>, String>>,
}

#[tokio::main]
async fn main() {
    let (req_tx, mut req_rx) = mpsc::channel::<RequêteBDD>(100);
    
    // Worker BDD (simule une connexion)
    tokio::spawn(async move {
        while let Some(req) = req_rx.recv().await {
            let résultat = exécuter_sql(&req.sql).await;
            let _ = req.réponse.send(résultat); // Envoyer la réponse
        }
    });
    
    // Envoyer une requête et attendre la réponse
    let (tx_réponse, rx_réponse) = oneshot::channel();
    
    req_tx.send(RequêteBDD {
        sql: "SELECT * FROM utilisateurs".to_string(),
        réponse: tx_réponse,
    }).await.unwrap();
    
    match rx_réponse.await {
        Ok(Ok(lignes)) => println!("{} lignes retournées", lignes.len()),
        Ok(Err(e)) => eprintln!("Erreur SQL : {}", e),
        Err(_) => eprintln!("Worker mort avant de répondre"),
    }
}

async fn exécuter_sql(sql: &str) -> Result<Vec<String>, String> {
    // Simulation
    tokio::time::sleep(std::time::Duration::from_millis(10)).await;
    Ok(vec!["ligne1".to_string(), "ligne2".to_string()])
}
```

## watch — Partager un état mutable observé

```rust
use tokio::sync::watch;

#[derive(Debug, Clone)]
enum ÉtatServeur {
    Démarrage,
    Prêt,
    Dégradé { raison: String },
    Arrêt,
}

#[tokio::main]
async fn main() {
    let (tx_état, rx_état) = watch::channel(ÉtatServeur::Démarrage);
    
    // Plusieurs observateurs
    let mut observer1 = rx_état.clone();
    let mut observer2 = rx_état.clone();
    
    // Tâche qui observe les changements d'état
    tokio::spawn(async move {
        loop {
            observer1.changed().await.unwrap();
            println!("Observer 1 : état = {:?}", *observer1.borrow());
        }
    });
    
    tokio::spawn(async move {
        loop {
            observer2.changed().await.unwrap();
            let état = observer2.borrow().clone();
            if matches!(état, ÉtatServeur::Arrêt) {
                println!("Observer 2 : arrêt détecté, nettoyage...");
                break;
            }
        }
    });
    
    // Simuler des transitions d'état
    tokio::time::sleep(std::time::Duration::from_millis(100)).await;
    tx_état.send(ÉtatServeur::Prêt).unwrap();
    
    tokio::time::sleep(std::time::Duration::from_millis(500)).await;
    tx_état.send(ÉtatServeur::Dégradé {
        raison: "Connexion BDD lente".to_string()
    }).unwrap();
    
    tokio::time::sleep(std::time::Duration::from_millis(200)).await;
    tx_état.send(ÉtatServeur::Arrêt).unwrap();
    
    tokio::time::sleep(std::time::Duration::from_millis(100)).await;
}
```

## Mutex async — Partage de données entre tâches

```rust
use tokio::sync::{Mutex, RwLock};
use std::sync::Arc;
use std::collections::HashMap;

type Cache = Arc<RwLock<HashMap<String, String>>>;

async fn obtenir_ou_charger(cache: &Cache, clé: &str) -> String {
    // D'abord essayer en lecture (pas de blocage exclusif)
    {
        let lecture = cache.read().await;
        if let Some(val) = lecture.get(clé) {
            return val.clone();
        }
    } // Le verrou de lecture est libéré ici
    
    // Charger depuis la source (sans tenir le verrou)
    let valeur = charger_depuis_db(clé).await;
    
    // Écrire dans le cache
    {
        let mut écriture = cache.write().await;
        écriture.insert(clé.to_string(), valeur.clone());
    }
    
    valeur
}

async fn charger_depuis_db(clé: &str) -> String {
    tokio::time::sleep(std::time::Duration::from_millis(50)).await;
    format!("valeur_pour_{}", clé)
}
```

## 🔥 Failure First — Déadlock async

Contrairement aux threads OS, un deadlock async ne fait pas crasher le programme — il se **suspend silencieusement**.

```rust
use tokio::sync::Mutex;
use std::sync::Arc;

#[tokio::main]
async fn main() {
    let mutex_a = Arc::new(Mutex::new(0));
    let mutex_b = Arc::new(Mutex::new(0));
    
    let a1 = Arc::clone(&mutex_a);
    let b1 = Arc::clone(&mutex_b);
    
    // Tâche 1 : prend A puis B
    let t1 = tokio::spawn(async move {
        let _guard_a = a1.lock().await;
        println!("T1 a pris A");
        tokio::time::sleep(std::time::Duration::from_millis(50)).await;
        let _guard_b = b1.lock().await; // ← DEADLOCK : T2 tient B
        println!("T1 a pris B"); // Ne s'affiche jamais
    });
    
    let a2 = Arc::clone(&mutex_a);
    let b2 = Arc::clone(&mutex_b);
    
    // Tâche 2 : prend B puis A (ordre inversé)
    let t2 = tokio::spawn(async move {
        let _guard_b = b2.lock().await;
        println!("T2 a pris B");
        tokio::time::sleep(std::time::Duration::from_millis(50)).await;
        let _guard_a = a2.lock().await; // ← DEADLOCK : T1 tient A
        println!("T2 a pris A"); // Ne s'affiche jamais
    });
    
    // Le programme se suspend ici indéfiniment — aucun crash, aucun log d'erreur
    let _ = tokio::join!(t1, t2);
}
```

## Pattern : Acteur simple

```rust
use tokio::sync::{mpsc, oneshot};

// Messages que l'acteur peut recevoir
enum MessageCompteur {
    Incrémenter,
    Décrémenter,
    Obtenir { réponse: oneshot::Sender<i64> },
}

// L'acteur — une tâche avec état isolé
struct Compteur {
    valeur: i64,
}

impl Compteur {
    async fn run(mut self, mut rx: mpsc::Receiver<MessageCompteur>) {
        while let Some(msg) = rx.recv().await {
            match msg {
                MessageCompteur::Incrémenter => self.valeur += 1,
                MessageCompteur::Décrémenter => self.valeur -= 1,
                MessageCompteur::Obtenir { réponse } => {
                    let _ = réponse.send(self.valeur);
                }
            }
        }
    }
}

// Handle pour communiquer avec l'acteur
#[derive(Clone)]
struct HandleCompteur(mpsc::Sender<MessageCompteur>);

impl HandleCompteur {
    fn démarrer() -> Self {
        let (tx, rx) = mpsc::channel(64);
        tokio::spawn(Compteur { valeur: 0 }.run(rx));
        HandleCompteur(tx)
    }
    
    async fn incrémenter(&self) {
        self.0.send(MessageCompteur::Incrémenter).await.unwrap();
    }
    
    async fn obtenir(&self) -> i64 {
        let (tx, rx) = oneshot::channel();
        self.0.send(MessageCompteur::Obtenir { réponse: tx }).await.unwrap();
        rx.await.unwrap()
    }
}

#[tokio::main]
async fn main() {
    let compteur = HandleCompteur::démarrer();
    
    // Utilisable depuis plusieurs tâches simultanément
    let c1 = compteur.clone();
    let c2 = compteur.clone();
    
    tokio::join!(
        async move { for _ in 0..100 { c1.incrémenter().await; } },
        async move { for _ in 0..50  { c2.incrémenter().await; } },
    );
    
    println!("Valeur finale : {}", compteur.obtenir().await); // 150
}
```

**Avantage clé :** L'état de l'acteur (`valeur`) n'est jamais partagé — il vit dans une seule tâche. Pas de Mutex, pas de risque de deadlock sur ces données.

## Résumé des channels

| Channel | Producteurs | Consommateurs | Cas d'usage |
|---|---|---|---|
| `mpsc` | Plusieurs | Un | File de travail, pipeline |
| `broadcast` | Un | Plusieurs | Événements, notifications |
| `oneshot` | Un | Un | Requête/réponse |
| `watch` | Un | Plusieurs | État partagé observable |
