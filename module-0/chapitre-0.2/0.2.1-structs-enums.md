## L'art de modéliser les données

La qualité d'un système distribué commence avant même d'écrire la logique métier — elle commence dans la manière dont vous modélisez vos données. Un bon modèle de données rend les états invalides **impossibles à représenter**, ce qui élimine toute une classe de bugs.

En Rust, les deux outils principaux pour cela sont les `struct` et les `enum`. Ensemble, ils constituent un système de types expressif capable d'encoder les invariants de votre domaine directement dans le compilateur.

## Structs — Regrouper des données liées

Une `struct` est un type composé qui regroupe plusieurs champs nommés.

### Struct classique

```rust
#[derive(Debug, Clone)]
struct Serveur {
    hote: String,
    port: u16,
    actif: bool,
    connexions_max: u32,
}

impl Serveur {
    // Constructeur associé (convention "new")
    fn new(hote: impl Into<String>, port: u16) -> Self {
        Serveur {
            hote: hote.into(),
            port,
            actif: false,
            connexions_max: 1000,
        }
    }
    
    // Méthode qui emprunte &self
    fn adresse(&self) -> String {
        format!("{}:{}", self.hote, self.port)
    }
    
    // Méthode qui modifie &mut self
    fn demarrer(&mut self) {
        self.actif = true;
        println!("Serveur {} démarré", self.adresse());
    }
    
    // Méthode qui consomme self (pattern builder)
    fn avec_connexions_max(mut self, max: u32) -> Self {
        self.connexions_max = max;
        self
    }
}

fn main() {
    // Pattern builder fluent
    let mut serveur = Serveur::new("localhost", 8080)
        .avec_connexions_max(500);
    
    serveur.demarrer();
    println!("{:?}", serveur);
}
```

### Struct Tuple

Utile pour créer des types distincts autour de types simples (newtype pattern) :

```rust
// Deux types distincts — évite de les confondre !
struct Metres(f64);
struct Secondes(f64);
struct Identifiant(u64);

// Maintenant le compilateur refuse de mélanger les types
fn vitesse(distance: Metres, temps: Secondes) -> f64 {
    distance.0 / temps.0
}

fn main() {
    let d = Metres(100.0);
    let t = Secondes(9.58);
    
    println!("Vitesse : {:.2} m/s", vitesse(d, t));
    
    // vitesse(t, d); ← ERREUR DE COMPILATION — types inversés impossible
}
```

### Struct Unitaire

Sans champs — utile pour les trait objects et les marqueurs :

```rust
struct MarqueurAdmin;
struct MarqueurLectureSeule;

// Ces types sont utilisés comme phantom types ou markers
```

## Enums — Modéliser des alternatives

Les `enum` en Rust sont bien plus puissantes que dans la plupart des langages — chaque variant peut contenir des données différentes.

### Enum simple

```rust
#[derive(Debug, PartialEq)]
enum Statut {
    Actif,
    Inactif,
    EnMaintenance,
    Erreur,
}
```

### Enum avec données (la vraie puissance)

```rust
#[derive(Debug)]
enum MessageReseau {
    // Variant sans données
    Ping,
    Pong,
    
    // Variant avec données simples
    EchoTexte(String),
    
    // Variant avec données typées multiples
    TransfertFichier {
        nom: String,
        taille: u64,
        contenu: Vec<u8>,
    },
    
    // Variant avec erreur
    Erreur {
        code: u32,
        message: String,
    },
}

fn traiter_message(msg: MessageReseau) {
    match msg {
        MessageReseau::Ping => println!("Reçu Ping, envoi Pong"),
        MessageReseau::Pong => println!("Pong reçu"),
        MessageReseau::EchoTexte(texte) => println!("Echo : {}", texte),
        MessageReseau::TransfertFichier { nom, taille, .. } => {
            println!("Fichier '{}' ({} octets)", nom, taille);
        }
        MessageReseau::Erreur { code, message } => {
            eprintln!("Erreur {}: {}", code, message);
        }
    }
}
```

## Pattern Matching — L'outil qui rend les enums utiles

Le `match` en Rust est exhaustif : vous devez couvrir **tous** les cas. Le compilateur refuse le code qui en oublie.

```rust
enum Direction {
    Nord,
    Sud,
    Est,
    Ouest,
}

fn deplacer(pos: (i32, i32), dir: Direction) -> (i32, i32) {
    let (x, y) = pos;
    match dir {
        Direction::Nord => (x, y + 1),
        Direction::Sud  => (x, y - 1),
        Direction::Est  => (x + 1, y),
        Direction::Ouest => (x - 1, y),
        // Pas de `_ =>` nécessaire — tous les cas sont couverts
        // Si on ajoute Direction::HautBas plus tard, le compilateur signalera ici
    }
}
```

### Guards et patterns complexes

```rust
fn classifier_port(port: u16) -> &'static str {
    match port {
        0 => "Réservé",
        1..=1023 => "Ports système",
        8080 | 8443 | 3000 => "Ports développement courants",
        1024..=49151 => "Ports enregistrés",
        _ => "Ports dynamiques/privés",
    }
}

// Matching sur des structs
struct Requete {
    methode: String,
    chemin: String,
    authentifié: bool,
}

fn router(req: &Requete) -> &'static str {
    match req {
        Requete { methode, chemin, authentifié: true }
            if methode == "GET" && chemin.starts_with("/admin") =>
        {
            "Admin OK"
        }
        Requete { methode, authentifié: false, .. }
            if methode == "GET" =>
        {
            "Lecture publique"
        }
        Requete { authentifié: false, .. } => "Non autorisé",
        _ => "Autre",
    }
}
```

## Le pattern "Make Illegal States Unrepresentable"

C'est l'un des patterns les plus importants en ingénierie logicielle. Voici comment l'appliquer en Rust :

### Mauvaise modélisation (états invalides possibles)

```rust
// ❌ Problème : actif=true et heure_deconnexion=Some(...) est incohérent
struct SessionUtilisateur {
    id: u64,
    actif: bool,                      // true ou false
    heure_connexion: Option<String>,  // None si jamais connecté
    heure_deconnexion: Option<String>,// None si encore connecté
}
```

### Bonne modélisation (états invalides impossibles)

```rust
// ✅ Chaque état est explicite et ne peut pas être incohérent
#[derive(Debug)]
enum EtatSession {
    JamaisConnecté,
    Connecté {
        heure_connexion: String,
    },
    Déconnecté {
        heure_connexion: String,
        heure_deconnexion: String,
    },
}

struct SessionUtilisateur {
    id: u64,
    etat: EtatSession,
}

impl SessionUtilisateur {
    fn connecter(&mut self, heure: String) {
        self.etat = match &self.etat {
            EtatSession::JamaisConnecté | EtatSession::Déconnecté { .. } => {
                EtatSession::Connecté { heure_connexion: heure }
            }
            EtatSession::Connecté { .. } => {
                // Déjà connecté — ignorer ou retourner une erreur
                return;
            }
        };
    }
    
    fn deconnecter(&mut self, heure: String) {
        self.etat = match std::mem::take(&mut self.etat) {
            EtatSession::Connecté { heure_connexion } => {
                EtatSession::Déconnecté { heure_connexion, heure_deconnexion: heure }
            }
            autre => autre, // Pas connecté — ignorer
        };
    }
}
```

## Dériver des traits automatiquement

```rust
#[derive(
    Debug,   // println!("{:?}", valeur)
    Clone,   // valeur.clone()
    PartialEq, Eq, // valeur1 == valeur2
    PartialOrd, Ord, // comparaison et tri
    Hash,    // utilisation comme clé HashMap
    Default, // Valeur::default() → valeur zéro
)]
struct ConfigReseau {
    hote: String,
    port: u16,
    timeout_ms: u32,
}

impl Default for ConfigReseau {
    fn default() -> Self {
        ConfigReseau {
            hote: "localhost".to_string(),
            port: 8080,
            timeout_ms: 5000,
        }
    }
}
```

## 🛠️ En production — Modélisation d'une réponse API

```rust
use std::collections::HashMap;

#[derive(Debug, serde::Serialize, serde::Deserialize)]
#[serde(tag = "type", content = "data")]
enum ReponseAPI<T> {
    #[serde(rename = "success")]
    Succès {
        données: T,
        méta: Option<MétaDonnées>,
    },
    #[serde(rename = "error")]
    Erreur {
        code: String,
        message: String,
        détails: Option<HashMap<String, String>>,
    },
    #[serde(rename = "paginated")]
    Paginée {
        données: Vec<T>,
        page: u32,
        total: u64,
    },
}

#[derive(Debug, serde::Serialize, serde::Deserialize)]
struct MétaDonnées {
    durée_ms: u64,
    version: String,
}

// Utilisation
fn obtenir_utilisateur(id: u64) -> ReponseAPI<Utilisateur> {
    match chercher_en_base(id) {
        Some(user) => ReponseAPI::Succès {
            données: user,
            méta: Some(MétaDonnées {
                durée_ms: 12,
                version: "v2".to_string(),
            }),
        },
        None => ReponseAPI::Erreur {
            code: "USER_NOT_FOUND".to_string(),
            message: format!("Utilisateur {} introuvable", id),
            détails: None,
        },
    }
}
```

## Résumé

| Outil | Usage | Avantage clé |
|---|---|---|
| `struct` | Regrouper des données liées | Organisation claire, méthodes associées |
| Newtype (`struct Metres(f64)`) | Distinctions sémantiques | Empêche les confusions de types |
| `enum` | Alternatives exclusives | Exhaustivité garantie par le compilateur |
| `match` | Traitement des variants | Impossible d'oublier un cas |
| `#[derive(...)]` | Traits automatiques | Moins de boilerplate |
