## Pourquoi généraliser son code ?

Imaginez que vous implémentez une fonction pour trouver le plus grand élément d'un `Vec<i32>`. Puis on vous demande la même chose pour `Vec<f64>`, puis pour `Vec<String>`. Copier-coller la logique en changeant juste le type, c'est une recette pour les bugs et la maintenance cauchemardesque.

Les **generics** et les **traits** sont la réponse de Rust à ce problème. Ensemble, ils permettent d'écrire du code générique qui fonctionne pour de nombreux types, sans sacrifier la performance.

## Les Traits — Définir des comportements

Un **trait** définit une interface : un ensemble de méthodes qu'un type doit implémenter.

```rust
// Définition du trait
trait Sérialisable {
    fn sérialiser(&self) -> String;
    fn désérialiser(s: &str) -> Self where Self: Sized;
    
    // Méthode avec implémentation par défaut
    fn taille_sérialisée(&self) -> usize {
        self.sérialiser().len()
    }
}

// Implémentation pour un type concret
struct Point {
    x: f64,
    y: f64,
}

impl Sérialisable for Point {
    fn sérialiser(&self) -> String {
        format!("{},{}", self.x, self.y)
    }
    
    fn désérialiser(s: &str) -> Self {
        let parts: Vec<&str> = s.split(',').collect();
        Point {
            x: parts[0].parse().unwrap(),
            y: parts[1].parse().unwrap(),
        }
    }
    // taille_sérialisée() est héritée automatiquement
}

fn main() {
    let p = Point { x: 3.0, y: 4.0 };
    let s = p.sérialiser();
    println!("Sérialisé : {}", s);
    println!("Taille : {}", p.taille_sérialisée());
    
    let p2 = Point::désérialiser(&s);
    println!("Désérialisé : ({}, {})", p2.x, p2.y);
}
```

### Traits courants de la bibliothèque standard

```rust
// Display : formatage pour l'utilisateur
use std::fmt;

impl fmt::Display for Point {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        write!(f, "({}, {})", self.x, self.y)
    }
}

// From/Into : conversions
impl From<(f64, f64)> for Point {
    fn from((x, y): (f64, f64)) -> Self {
        Point { x, y }
    }
}

fn main() {
    let p: Point = (3.0, 4.0).into(); // Utilise From automatiquement
    println!("{}", p); // Utilise Display
}
```

## Les Generics — Paramétrer par le type

```rust
// Fonction générique : fonctionne pour tout T qui implémente PartialOrd
fn maximum<T: PartialOrd>(liste: &[T]) -> Option<&T> {
    if liste.is_empty() {
        return None;
    }
    
    let mut max = &liste[0];
    for item in &liste[1..] {
        if item > max {
            max = item;
        }
    }
    Some(max)
}

fn main() {
    let entiers = vec![34, 50, 25, 100, 65];
    println!("Maximum entiers : {}", maximum(&entiers).unwrap());
    
    let flottants = vec![34.0, 50.0, 25.0, 100.0];
    println!("Maximum flottants : {}", maximum(&flottants).unwrap());
    
    let textes = vec!["bonjour", "monde", "rust"];
    println!("Maximum texte : {}", maximum(&textes).unwrap());
}
```

### Structs génériques

```rust
#[derive(Debug)]
struct Résultat<T, E> {
    valeur: Option<T>,
    erreur: Option<E>,
    durée_ms: u64,
}

impl<T, E> Résultat<T, E> {
    fn succès(valeur: T, durée_ms: u64) -> Self {
        Résultat { valeur: Some(valeur), erreur: None, durée_ms }
    }
    
    fn échec(erreur: E, durée_ms: u64) -> Self {
        Résultat { valeur: None, erreur: Some(erreur), durée_ms }
    }
    
    fn est_ok(&self) -> bool {
        self.valeur.is_some()
    }
}
```

## Bounds de traits — Contraindre les generics

Pour utiliser les méthodes d'un type générique, vous devez déclarer quelles interfaces il implémente :

```rust
use std::fmt::Display;

// Syntaxe "trait bound" classique
fn afficher_max<T: PartialOrd + Display>(a: T, b: T) {
    if a > b {
        println!("Maximum : {}", a);
    } else {
        println!("Maximum : {}", b);
    }
}

// Syntaxe "where" — plus lisible pour les bounds complexes
fn traiter_et_afficher<T, E>(résultat: Result<T, E>)
where
    T: Display + Clone,
    E: Display + std::error::Error,
{
    match résultat {
        Ok(val) => println!("Succès : {}", val),
        Err(e) => eprintln!("Erreur : {}", e),
    }
}
```

## Trait Objects vs Generics — Un choix architectural important

C'est l'une des décisions les plus importantes en Rust. Voyons la différence :

### Generics — Dispatch statique (monomorphisation)

```rust
trait Compression {
    fn compresser(&self, données: &[u8]) -> Vec<u8>;
}

struct Gzip;
struct Zstd;

impl Compression for Gzip {
    fn compresser(&self, données: &[u8]) -> Vec<u8> {
        // Implémentation Gzip...
        données.to_vec() // Simplification
    }
}

impl Compression for Zstd {
    fn compresser(&self, données: &[u8]) -> Vec<u8> {
        // Implémentation Zstd...
        données.to_vec() // Simplification
    }
}

// Générique — le compilateur génère du code spécifique pour Gzip et Zstd
fn pipeline_générique<C: Compression>(compresseur: &C, données: &[u8]) -> Vec<u8> {
    println!("Compression de {} octets", données.len());
    compresseur.compresser(données)
}
```

**Avantages :** Zéro overhead à l'exécution — le compilateur génère du code optimisé pour chaque type concret.
**Inconvénients :** Taille du binaire plus grande (une version par type), le type doit être connu à la compilation.

### Trait Objects — Dispatch dynamique

```rust
// &dyn Trait ou Box<dyn Trait> — résolution à l'exécution
fn pipeline_dynamique(compresseur: &dyn Compression, données: &[u8]) -> Vec<u8> {
    println!("Compression de {} octets", données.len());
    compresseur.compresser(données)
}

fn main() {
    let données = b"Hello, World!";
    
    // Générique : type connu à la compilation
    let gzip = Gzip;
    pipeline_générique(&gzip, données);
    
    // Dynamique : type décidé à l'exécution
    let compresseurs: Vec<Box<dyn Compression>> = vec![
        Box::new(Gzip),
        Box::new(Zstd),
    ];
    
    for c in &compresseurs {
        pipeline_dynamique(c.as_ref(), données);
    }
}
```

**Avantages :** Flexibilité maximale — liste hétérogène de types, plugin dynamique.
**Inconvénients :** Overhead vtable (une indirection), pas d'optimisation inline possible.

### Règle de décision

```
Q: Le type est-il connu à la compilation ?
├─> Oui : utilisez les Generics
└─> Non (liste hétérogène, config runtime, plugins) : utilisez Box<dyn Trait>

Q: Avez-vous besoin de performances maximales sur le chemin chaud ?
├─> Oui : Generics
└─> Non (traitement occasionnel) : les deux sont acceptables
```

## Traits avancés — Méthodes associées et types associés

```rust
trait Parseur {
    // Type associé : défini par l'implémenteur
    type Sortie;
    type Erreur;
    
    fn parser(&self, entrée: &str) -> Result<Self::Sortie, Self::Erreur>;
}

struct ParsurJSON;

impl Parseur for ParsurJSON {
    type Sortie = serde_json::Value;
    type Erreur = serde_json::Error;
    
    fn parser(&self, entrée: &str) -> Result<Self::Sortie, Self::Erreur> {
        serde_json::from_str(entrée)
    }
}
```

## 🔥 Failure First — L'erreur d'objet de trait trop large

```rust
// ❌ Ce code ne compile pas
trait Clone { // Supposons qu'on définisse notre propre Clone
    fn clone(&self) -> Self; // Retourne Self — taille inconnue pour dyn
}

// fn mauvaise<T>(liste: Vec<Box<dyn Clone>>) {} // ← Erreur : Clone n'est pas "object-safe"
```

Un trait est **object-safe** si ses méthodes ne retournent pas `Self` et n'utilisent pas de generics. C'est la condition pour pouvoir utiliser `dyn Trait`.

```rust
// ✅ Correction : utiliser une méthode de clonage avec Box
trait ClonableBoxed {
    fn clone_boxed(&self) -> Box<dyn ClonableBoxed>;
}
```

## 🛠️ Pattern production : Middleware générique

```rust
trait Middleware {
    fn traiter(&self, req: &mut Requete, suivant: &dyn Fn(&mut Requete));
}

struct Logger;
struct AuthMiddleware { token_secret: String }

impl Middleware for Logger {
    fn traiter(&self, req: &mut Requete, suivant: &dyn Fn(&mut Requete)) {
        println!("[LOG] {} {}", req.méthode, req.chemin);
        suivant(req);
        println!("[LOG] Réponse envoyée");
    }
}

impl Middleware for AuthMiddleware {
    fn traiter(&self, req: &mut Requete, suivant: &dyn Fn(&mut Requete)) {
        if req.headers.get("Authorization").is_some() {
            suivant(req);
        } else {
            req.statut = 401;
        }
    }
}

struct Pipeline {
    middlewares: Vec<Box<dyn Middleware>>,
}

impl Pipeline {
    fn ajouter(mut self, m: impl Middleware + 'static) -> Self {
        self.middlewares.push(Box::new(m));
        self
    }
}
```

## Résumé

| Concept | Quand l'utiliser |
|---|---|
| `trait` | Définir une interface partagée entre types |
| `impl Trait for Type` | Donner un comportement à un type |
| `fn f<T: Trait>` | Code générique avec dispatch statique |
| `fn f(x: &dyn Trait)` | Polymorphisme dynamique à l'exécution |
| Types associés | Quand le trait a un type "résultat" naturel |
| `where` clause | Bounds complexes, meilleure lisibilité |
