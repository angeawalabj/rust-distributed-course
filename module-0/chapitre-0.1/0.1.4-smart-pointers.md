## Quand l'ownership simple ne suffit pas

Les règles d'ownership que nous avons vues fonctionnent parfaitement pour 80% des cas. Mais certaines structures de données nécessitent des schémas plus flexibles :

- Un nœud de graphe référencé par plusieurs autres nœuds (ownership partagé)
- Une configuration partagée entre plusieurs threads (ownership partagé + thread-safety)
- Une valeur modifiée derrière une référence immutable (mutabilité intérieure)

Rust propose plusieurs **smart pointers** pour couvrir ces cas, chacun avec un coût et des garanties précis.

---

## Box\<T\> — L'allocation heap explicite

Déjà vu en 0.1.1, `Box<T>` est le smart pointer le plus simple. Il déplace une valeur sur le heap et en devient l'unique propriétaire.

### Cas d'usage principaux

**1. Types récursifs :**
```rust
// Sans Box : le compilateur refuse (taille infinie)
enum ArbreJson {
    Null,
    Booleen(bool),
    Nombre(f64),
    Texte(String),
    Tableau(Vec<ArbreJson>),          // OK — Vec connaît sa taille
    Objet(Vec<(String, ArbreJson)>),  // OK — Vec connaît sa taille
}

// Ou pour une liste chaînée classique :
enum Liste {
    Element(i32, Box<Liste>), // Box brise la récursion
    Fin,
}

fn main() {
    let liste = Liste::Element(1,
        Box::new(Liste::Element(2,
            Box::new(Liste::Element(3,
                Box::new(Liste::Fin))))));
}
```

**2. Trait objects — polymorphisme dynamique :**
```rust
trait Animal {
    fn cri(&self) -> &str;
}

struct Chien;
struct Chat;

impl Animal for Chien { fn cri(&self) -> &str { "Wouf" } }
impl Animal for Chat  { fn cri(&self) -> &str { "Miaou" } }

fn main() {
    // Vec de trait objects — taille inconnue à la compile → Box obligatoire
    let animaux: Vec<Box<dyn Animal>> = vec![
        Box::new(Chien),
        Box::new(Chat),
        Box::new(Chien),
    ];
    
    for animal in &animaux {
        println!("{}", animal.cri());
    }
}
```

**Coût de Box\<T\> :** une allocation heap + un niveau d'indirection. Pas de surcoût à la libération.

## Rc\<T\> — Ownership partagé (single-thread)

`Rc<T>` (Reference Counted) permet à plusieurs propriétaires de partager la même valeur. Un compteur interne est incrémenté à chaque clone, décrémenté à chaque drop. Quand le compteur atteint zéro, la valeur est libérée.

```rust
use std::rc::Rc;

fn main() {
    let valeur = Rc::new(String::from("partagée"));
    
    println!("Compteur de références : {}", Rc::strong_count(&valeur)); // 1
    
    let clone1 = Rc::clone(&valeur); // Incrémente le compteur — ne copie pas les données
    let clone2 = Rc::clone(&valeur);
    
    println!("Compteur : {}", Rc::strong_count(&valeur)); // 3
    
    drop(clone1);
    println!("Compteur après drop : {}", Rc::strong_count(&valeur)); // 2
    
    println!("{}", valeur);  // ✅
    println!("{}", clone2);  // ✅
} // valeur et clone2 droppés → compteur = 0 → libération
```

> ⚠️ **Limitation critique : `Rc<T>` n'est pas thread-safe.** Il ne peut être utilisé que dans un seul thread. Le compilateur vous empêchera d'envoyer un `Rc` vers un autre thread.

### Exemple concret : graphe de nœuds partagés

```rust
use std::rc::Rc;
use std::cell::RefCell;

type Noeud = Rc<RefCell<DonnéesNoeud>>;

struct DonnéesNoeud {
    valeur: i32,
    enfants: Vec<Noeud>,
}

fn main() {
    let racine = Rc::new(RefCell::new(DonnéesNoeud {
        valeur: 1,
        enfants: vec![],
    }));
    
    let enfant = Rc::new(RefCell::new(DonnéesNoeud {
        valeur: 2,
        enfants: vec![],
    }));
    
    // enfant est partagé par deux parents
    racine.borrow_mut().enfants.push(Rc::clone(&enfant));
    
    println!("Racine valeur : {}", racine.borrow().valeur);
    println!("Références vers enfant : {}", Rc::strong_count(&enfant)); // 2
}
```

## Arc\<T\> — Ownership partagé et thread-safe

`Arc<T>` (Atomically Reference Counted) est la version thread-safe de `Rc<T>`. Le compteur est incrémenté/décrémenté avec des opérations atomiques, ce qui a un léger coût mais garantit la sécurité multi-thread.

```rust
use std::sync::Arc;
use std::thread;

fn main() {
    let données = Arc::new(vec![1, 2, 3, 4, 5]);
    
    let mut handles = vec![];
    
    for i in 0..3 {
        let données_clone = Arc::clone(&données); // Clone l'Arc, pas les données
        
        let handle = thread::spawn(move || {
            println!("Thread {} voit : {:?}", i, données_clone);
        });
        
        handles.push(handle);
    }
    
    for handle in handles {
        handle.join().unwrap();
    }
    
    // données est toujours valide ici
    println!("Main voit : {:?}", données);
}
```

> 💡 **Règle pratique :** Utilisez `Rc` en single-thread (plus rapide), `Arc` en multi-thread. Le compilateur vous dira si vous vous trompez.

## Mutex\<T\> — Mutabilité thread-safe

`Arc<T>` donne un accès en lecture partagé. Pour écrire, il faut un `Mutex<T>` (ou `RwLock<T>`).

Un `Mutex` (mutual exclusion) garantit qu'un seul thread peut accéder aux données à la fois.

```rust
use std::sync::{Arc, Mutex};
use std::thread;

fn main() {
    // Compteur partagé entre plusieurs threads
    let compteur = Arc::new(Mutex::new(0u64));
    
    let mut handles = vec![];
    
    for _ in 0..10 {
        let compteur_clone = Arc::clone(&compteur);
        
        let handle = thread::spawn(move || {
            // lock() bloque jusqu'à ce que le mutex soit disponible
            // Retourne un MutexGuard — le verrou est libéré quand guard est droppé
            let mut guard = compteur_clone.lock().unwrap();
            *guard += 1;
            // ← guard droppé ici, verrou libéré
        });
        
        handles.push(handle);
    }
    
    for handle in handles {
        handle.join().unwrap();
    }
    
    println!("Compteur final : {}", *compteur.lock().unwrap()); // 10
}
```

### RwLock\<T\> — Lectures parallèles, écritures exclusives

```rust
use std::sync::{Arc, RwLock};

fn main() {
    let config = Arc::new(RwLock::new(std::collections::HashMap::new()));
    
    // Écriture : verrou exclusif
    {
        let mut write_guard = config.write().unwrap();
        write_guard.insert("hote", "localhost");
        write_guard.insert("port", "8080");
    }
    
    // Plusieurs lectures simultanées : OK avec RwLock
    let c1 = Arc::clone(&config);
    let c2 = Arc::clone(&config);
    
    let t1 = std::thread::spawn(move || {
        let read_guard = c1.read().unwrap();
        println!("Thread 1 - hote: {}", read_guard.get("hote").unwrap());
    });
    
    let t2 = std::thread::spawn(move || {
        let read_guard = c2.read().unwrap();
        println!("Thread 2 - port: {}", read_guard.get("port").unwrap());
    });
    
    t1.join().unwrap();
    t2.join().unwrap();
}
```

## 🔥 Failure First — Le deadlock

Un **deadlock** se produit quand deux threads attendent chacun un verrou détenu par l'autre.

```rust
use std::sync::{Arc, Mutex};
use std::thread;
use std::time::Duration;

fn main() {
    let mutex_a = Arc::new(Mutex::new(0));
    let mutex_b = Arc::new(Mutex::new(0));
    
    let a1 = Arc::clone(&mutex_a);
    let b1 = Arc::clone(&mutex_b);
    
    // Thread 1 : prend A puis B
    let t1 = thread::spawn(move || {
        let _guard_a = a1.lock().unwrap();
        println!("Thread 1 a pris A");
        thread::sleep(Duration::from_millis(100)); // Laisse le temps à T2 de prendre B
        println!("Thread 1 essaie de prendre B...");
        let _guard_b = b1.lock().unwrap(); // ← BLOQUÉ — T2 tient B
        println!("Thread 1 a pris B"); // Ne s'affichera jamais
    });
    
    let a2 = Arc::clone(&mutex_a);
    let b2 = Arc::clone(&mutex_b);
    
    // Thread 2 : prend B puis A (ordre inversé = deadlock)
    let t2 = thread::spawn(move || {
        let _guard_b = b2.lock().unwrap();
        println!("Thread 2 a pris B");
        thread::sleep(Duration::from_millis(100));
        println!("Thread 2 essaie de prendre A...");
        let _guard_a = a2.lock().unwrap(); // ← BLOQUÉ — T1 tient A
        println!("Thread 2 a pris A"); // Ne s'affichera jamais
    });
    
    t1.join().unwrap(); // Bloqué indéfiniment
    t2.join().unwrap(); // Idem
}
```

**Correction :** Toujours acquérir les verrous dans le **même ordre** dans tous les threads :

```rust
// Solution : les deux threads prennent toujours A avant B
let _guard_a = mutex_a.lock().unwrap();
let _guard_b = mutex_b.lock().unwrap();
// Maintenant il n'y a pas de cycle possible
```

## RefCell\<T\> — Mutabilité intérieure (single-thread)

`RefCell<T>` permet de muter des données derrière une référence immutable, avec vérification à l'exécution (et non à la compilation).

```rust
use std::cell::RefCell;

struct Noeud {
    valeur: i32,
    cache: RefCell<Option<i32>>, // Peut être muté même si Noeud est immutable
}

impl Noeud {
    fn valeur_calculée(&self) -> i32 {
        // self est &self (immutable), mais on peut muter le cache
        let mut cache = self.cache.borrow_mut();
        if cache.is_none() {
            *cache = Some(self.valeur * 2); // Calcul coûteux mis en cache
        }
        cache.unwrap()
    }
}
```

> ⚠️ Si vous appelez `borrow_mut()` sur un `RefCell` déjà emprunté mutuellement, le programme **panique à l'exécution**. Les règles du borrow checker s'appliquent toujours — elles sont juste vérifiées plus tard.

## Choisir le bon smart pointer

```
Besoin d'allouer sur le heap ?
└─> Box<T>

Besoin de partager l'ownership ?
├─> Single-thread : Rc<T>
└─> Multi-thread  : Arc<T>

Besoin de mutabilité partagée ?
├─> Single-thread : Rc<RefCell<T>>
└─> Multi-thread  : Arc<Mutex<T>> ou Arc<RwLock<T>>

Besoin de polymorphisme dynamique ?
└─> Box<dyn Trait>
```

## 🛠️ Pattern production : état partagé dans un serveur web

```rust
use std::sync::{Arc, RwLock};
use std::collections::HashMap;

// État de l'application partagé entre tous les handlers
#[derive(Clone)]
struct AppState {
    sessions: Arc<RwLock<HashMap<String, String>>>,
    compteur_requetes: Arc<std::sync::atomic::AtomicU64>,
}

impl AppState {
    fn nouvelle() -> Self {
        AppState {
            sessions: Arc::new(RwLock::new(HashMap::new())),
            compteur_requetes: Arc::new(std::sync::atomic::AtomicU64::new(0)),
        }
    }
    
    fn ajouter_session(&self, token: String, utilisateur: String) {
        self.sessions.write().unwrap().insert(token, utilisateur);
    }
    
    fn trouver_session(&self, token: &str) -> Option<String> {
        self.sessions.read().unwrap().get(token).cloned()
    }
    
    fn incrementer_compteur(&self) -> u64 {
        use std::sync::atomic::Ordering;
        self.compteur_requetes.fetch_add(1, Ordering::Relaxed)
    }
}
```

## Résumé

| Type | Thread-safe | Coût | Cas d'usage |
|---|---|---|---|
| `Box<T>` | N/A | Minimal | Heap, types récursifs, trait objects |
| `Rc<T>` | ❌ | Faible | Ownership partagé, single-thread |
| `Arc<T>` | ✅ | Modéré | Ownership partagé, multi-thread |
| `Mutex<T>` | ✅ | Blocage possible | Écriture exclusive |
| `RwLock<T>` | ✅ | Modéré | Lectures parallèles, écritures exclusives |
| `RefCell<T>` | ❌ | Vérif. runtime | Mutabilité intérieure, single-thread |
