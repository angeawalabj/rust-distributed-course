## Apprendre en cassant

Ce chapitre est différent des précédents. Nous n'allons pas construire quelque chose qui fonctionne d'emblée. Nous allons délibérément provoquer des erreurs, observer ce qu'elles produisent, et apprendre à les diagnostiquer avec méthode.

Un développeur qui n'a jamais vu un panic en production ne sait pas vraiment comment réagir quand ça arrive. Ce chapitre vous donne cet entraînement dans un environnement sûr.

## Les paniques en Rust

Rust distingue deux types d'erreurs :

- **Les erreurs récupérables :** `Result<T, E>` — attendues, gérées explicitement.
- **Les erreurs irrécupérables :** `panic!` — programme dans un état impossible, arrêt forcé.

```rust
fn main() {
    // Quelques façons de provoquer un panic
    
    // 1. Macro panic! directe
    // panic!("Quelque chose d'impossible s'est produit");
    
    // 2. Accès hors-limites
    let v = vec![1, 2, 3];
    // let _ = v[10]; // ← panic: index out of bounds: the len is 3 but the index is 10
    
    // 3. Unwrap sur None ou Err
    let option: Option<i32> = None;
    // let _ = option.unwrap(); // ← panic: called `Option::unwrap()` on a `None` value
    
    // 4. Division par zéro (entiers)
    let x = 0u32;
    // let _ = 5 / x; // ← panic: attempt to divide by zero
    
    println!("Ce message s'affiche si on commente les panics ci-dessus");
}
```

### Anatomie d'un message de panic

```
thread 'main' panicked at 'index out of bounds: the len is 3 but the index is 10', src/main.rs:8:13
note: run with `RUST_BACKTRACE=1` environment variable to display a backtrace
```

Ce message vous donne :
- Le thread qui a paniqué
- Le message d'erreur
- Le fichier et la ligne
- Comment obtenir plus d'informations

## 🔥 Exercice 1 — Activer et lire les backtraces

Créez ce fichier `src/main.rs` :

```rust
fn niveau_3() {
    let v: Vec<i32> = vec![];
    let _ = v[0]; // Panic ici
}

fn niveau_2() {
    niveau_3();
}

fn niveau_1() {
    niveau_2();
}

fn main() {
    niveau_1();
}
```

Exécutez avec backtrace :

```bash
RUST_BACKTRACE=1 cargo run
```

Vous verrez quelque chose comme :

```
thread 'main' panicked at 'index out of bounds: the len is 0 but the index is 0'

stack backtrace:
   0: rust_begin_unwind
   1: core::panicking::panic_fmt
   2: core::slice::index::slice_index_usize_fail
   3: mon_projet::niveau_3
        at ./src/main.rs:3:13
   4: mon_projet::niveau_2
        at ./src/main.rs:7:5
   5: mon_projet::niveau_1
        at ./src/main.rs:11:5
   6: mon_projet::main
        at ./src/main.rs:15:5
```

**Ce que vous apprenez à lire :**
- Les frames du bas vers le haut : `main` → `niveau_1` → `niveau_2` → `niveau_3`
- La ligne exacte du panic : `src/main.rs:3:13`
- Les frames système (ignorez-les généralement)

Avec `RUST_BACKTRACE=full`, vous obtenez encore plus de détails incluant les adresses mémoire.

## 🔥 Exercice 2 — Simuler une fuite mémoire avec Rc

Rust empêche les fuites mémoire dans la plupart des cas — mais `Rc` + cycles de références est un cas où elles peuvent survenir :

```rust
use std::rc::Rc;
use std::cell::RefCell;

#[derive(Debug)]
struct Noeud {
    valeur: i32,
    voisin: RefCell<Option<Rc<Noeud>>>,
}

impl Drop for Noeud {
    fn drop(&mut self) {
        println!("Noeud {} droppé", self.valeur);
    }
}

fn main() {
    let a = Rc::new(Noeud {
        valeur: 1,
        voisin: RefCell::new(None),
    });
    
    let b = Rc::new(Noeud {
        valeur: 2,
        voisin: RefCell::new(None),
    });
    
    // Créer un cycle : a pointe vers b, b pointe vers a
    *a.voisin.borrow_mut() = Some(Rc::clone(&b));
    *b.voisin.borrow_mut() = Some(Rc::clone(&a));
    
    println!("Compteur a : {}", Rc::strong_count(&a)); // 2
    println!("Compteur b : {}", Rc::strong_count(&b)); // 2
    
    // Fin de main : a et b sont droppés
    // Mais les compteurs vont à 1 (pas 0) à cause du cycle
    // → Le Drop n'est jamais appelé → fuite mémoire
}
```

Sortie : les messages "Noeud X droppé" **ne s'affichent pas** — preuve de la fuite.

**Correction avec `Weak<T>` :**

```rust
use std::rc::{Rc, Weak};
use std::cell::RefCell;

#[derive(Debug)]
struct Noeud {
    valeur: i32,
    voisin: RefCell<Option<Weak<Noeud>>>, // Weak ne compte pas dans strong_count
}

impl Drop for Noeud {
    fn drop(&mut self) {
        println!("Noeud {} droppé proprement", self.valeur);
    }
}

fn main() {
    let a = Rc::new(Noeud { valeur: 1, voisin: RefCell::new(None) });
    let b = Rc::new(Noeud { valeur: 2, voisin: RefCell::new(None) });
    
    // Weak reference : ne maintient pas en vie
    *a.voisin.borrow_mut() = Some(Rc::downgrade(&b));
    *b.voisin.borrow_mut() = Some(Rc::downgrade(&a));
    
    println!("Strong count a : {}", Rc::strong_count(&a)); // 1
    println!("Strong count b : {}", Rc::strong_count(&b)); // 1
}
// Les deux noeuds sont droppés correctement
```

## 🔥 Exercice 3 — Panic dans un thread

Comment Rust gère-t-il un panic dans un thread secondaire ?

```rust
use std::thread;

fn main() {
    let handle = thread::spawn(|| {
        println!("Thread démarré");
        
        let v: Vec<i32> = vec![1, 2, 3];
        println!("{}", v[10]); // Panic ici
    });
    
    // join() retourne Err si le thread a paniqué
    match handle.join() {
        Ok(_) => println!("Thread terminé normalement"),
        Err(e) => {
            // e est Box<dyn Any> — on peut tenter d'extraire le message
            if let Some(s) = e.downcast_ref::<&str>() {
                println!("Thread paniqué avec : {}", s);
            } else if let Some(s) = e.downcast_ref::<String>() {
                println!("Thread paniqué avec : {}", s);
            } else {
                println!("Thread paniqué (type inconnu)");
            }
        }
    }
    
    println!("Le programme principal continue malgré le panic du thread");
}
```

**Leçon clé :** Un panic dans un thread secondaire ne fait pas crasher les autres threads. C'est essentiel pour les serveurs : un handler de requête qui panique ne fait pas tomber tout le serveur.

## Intercepter les panics avec `catch_unwind`

Dans certains cas (serveurs, FFI), vous voulez capturer un panic sans terminer le programme :

```rust
use std::panic;

fn operation_risquee(n: i32) -> i32 {
    if n == 0 {
        panic!("Division par zéro !");
    }
    100 / n
}

fn main() {
    // catch_unwind intercepte le panic
    let résultat = panic::catch_unwind(|| {
        operation_risquee(0)
    });
    
    match résultat {
        Ok(val) => println!("Succès : {}", val),
        Err(_) => println!("Panic intercepté — on continue"),
    }
    
    // Le programme continue
    let résultat2 = panic::catch_unwind(|| operation_risquee(5));
    println!("Deuxième essai : {:?}", résultat2); // Ok(20)
}
```

> ⚠️ `catch_unwind` ne devrait **pas** être utilisé pour la gestion d'erreurs ordinaires — c'est `Result` pour ça. Réservez `catch_unwind` aux frontières critiques (serveurs de requête, plugins WASM).

## Outils de diagnostic mémoire

### Valgrind (Linux)

```bash
# Compiler en mode debug avec symboles
cargo build

# Analyser avec Valgrind
valgrind --leak-check=full --show-leak-kinds=all ./target/debug/mon_programme
```

En pratique avec Rust, Valgrind ne trouve presque jamais de fuites (c'est le but) — mais il peut détecter des bugs dans du code unsafe.

### MIRI — L'interpréteur de Rust unsafe

```bash
# Installer MIRI
rustup component add miri

# Analyser votre code
cargo miri run
cargo miri test
```

MIRI détecte les Undefined Behaviors dans le code `unsafe` — indispensable si vous écrivez des abstractions bas niveau.

### Heaptrack — Profiling mémoire

```bash
# Linux
sudo apt install heaptrack

heaptrack ./target/debug/mon_programme
heaptrack_gui heaptrack.mon_programme.*.zst
```

## 🔥 Exercice Final — Debug d'une "panique mystérieuse"

Voici un code avec un bug subtil. Trouvez et corrigez le problème :

```rust
use std::collections::HashMap;

fn compter_mots(texte: &str) -> HashMap<&str, usize> {
    let mut compteur = HashMap::new();
    for mot in texte.split_whitespace() {
        let compte = compteur.entry(mot).or_insert(0);
        *compte += 1;
    }
    compteur
}

fn top_3(compteur: &HashMap<&str, usize>) -> Vec<(&str, usize)> {
    let mut paires: Vec<_> = compteur.iter().map(|(&k, &v)| (k, v)).collect();
    paires.sort_by(|a, b| b.1.cmp(&a.1));
    paires[..3].to_vec() // ← Bug potentiel ici
}

fn main() {
    let texte = "bonjour monde bonjour rust";
    let compteur = compter_mots(texte);
    
    // Ce code va paniquer si le texte a moins de 3 mots uniques !
    let top = top_3(&compteur);
    for (mot, count) in top {
        println!("{}: {}", mot, count);
    }
}
```

**Diagnostic :**

```bash
RUST_BACKTRACE=1 cargo run
# thread 'main' panicked at 'range end index 3 out of range for slice of length 2'
```

**Correction :**

```rust
fn top_3(compteur: &HashMap<&str, usize>) -> Vec<(&str, usize)> {
    let mut paires: Vec<_> = compteur.iter().map(|(&k, &v)| (k, v)).collect();
    paires.sort_by(|a, b| b.1.cmp(&a.1));
    
    let limite = paires.len().min(3); // Jamais plus que ce qu'on a
    paires[..limite].to_vec()
}
```

## Bonnes pratiques anti-panic en production

```rust
// ❌ Éviter : unwrap() en production
let valeur = option.unwrap();

// ✅ Préférer : gérer l'absence explicitement
let valeur = option.unwrap_or_default();
let valeur = option.unwrap_or(42);
let valeur = option.expect("Cette valeur doit exister — si non, c'est un bug critique");

// ✅ Encore mieux : propager l'erreur
fn ma_fonction() -> Result<Valeur, MonErreur> {
    let valeur = option.ok_or(MonErreur::ValeurManquante)?;
    Ok(valeur)
}

// ❌ Éviter : indexation directe sur des données utilisateur
let premier = ma_liste[0];

// ✅ Préférer :
let premier = ma_liste.first().ok_or(MonErreur::ListeVide)?;
// ou
if let Some(premier) = ma_liste.first() {
    // ...
}
```

## Résumé

| Erreur | Détection | Correction |
|---|---|---|
| Panic | RUST_BACKTRACE=1 | Remplacer unwrap() par gestion explicite |
| Cycle Rc | Le Drop n'est pas appelé | Utiliser Weak pour les références arrières |
| Deadlock | Programme bloqué | Ordre fixe des verrous |
| Fuite (unsafe) | MIRI, Valgrind | Vérifier la logique unsafe |
| Accès hors-limites | Backtrace clair | Utiliser .get() ou .first() |
