## La structure d'un projet Rust professionnel

Un projet Rust mal organisé devient rapidement ingérable. Voici la structure recommandée pour un projet backend de taille réelle :

```
mon-service/
├── Cargo.toml              ← Manifest principal
├── Cargo.lock              ← Versions fixées (à committer pour les binaires)
├── .cargo/
│   └── config.toml         ← Configuration cargo locale
├── src/
│   ├── main.rs             ← Point d'entrée
│   ├── lib.rs              ← Bibliothèque réexportée (pour les tests d'intégration)
│   ├── config.rs           ← Configuration et variables d'environnement
│   ├── erreurs.rs          ← Types d'erreurs centralisés
│   ├── modèles/
│   │   ├── mod.rs
│   │   ├── utilisateur.rs
│   │   └── session.rs
│   ├── handlers/           ← Handlers HTTP
│   │   ├── mod.rs
│   │   ├── auth.rs
│   │   └── utilisateurs.rs
│   ├── services/           ← Logique métier
│   │   ├── mod.rs
│   │   └── auth_service.rs
│   └── db/                 ← Couche données
│       ├── mod.rs
│       └── repo_utilisateurs.rs
├── tests/
│   ├── commun/
│   │   └── mod.rs
│   └── intégration.rs
├── benches/
│   └── performance.rs
└── migrations/             ← Migrations SQL (sqlx)
    ├── 001_init.sql
    └── 002_add_sessions.sql
```

## Workspaces Cargo — Monorepo pour projets complexes

Un workspace permet de gérer plusieurs crates liées dans un seul dépôt :

```
mon-système-distribué/
├── Cargo.toml              ← Workspace root
├── crates/
│   ├── serveur-api/        ← Binaire : API REST
│   │   ├── Cargo.toml
│   │   └── src/main.rs
│   ├── serveur-worker/     ← Binaire : worker de tâches
│   │   ├── Cargo.toml
│   │   └── src/main.rs
│   ├── protocole/          ← Bibliothèque : types et sérialization partagés
│   │   ├── Cargo.toml
│   │   └── src/lib.rs
│   └── commun/             ← Bibliothèque : utilitaires partagés
│       ├── Cargo.toml
│       └── src/lib.rs
```

```toml
# Cargo.toml à la racine du workspace
[workspace]
members = [
    "crates/serveur-api",
    "crates/serveur-worker",
    "crates/protocole",
    "crates/commun",
]
resolver = "2"

# Dépendances partagées — évite les doublons de versions
[workspace.dependencies]
tokio = { version = "1", features = ["full"] }
serde = { version = "1", features = ["derive"] }
tracing = "0.1"
thiserror = "1"
```

```toml
# crates/serveur-api/Cargo.toml
[package]
name = "serveur-api"
version = "0.1.0"
edition = "2021"

[dependencies]
# Réutilisation des versions du workspace
tokio.workspace = true
serde.workspace = true
# Dépendances locales du workspace
protocole = { path = "../protocole" }
commun = { path = "../commun" }
# Dépendances spécifiques à cette crate
axum = "0.7"
```

```bash
# Compiler tout le workspace
cargo build --workspace

# Lancer les tests de toutes les crates
cargo test --workspace

# Cibler une crate spécifique
cargo run -p serveur-api
cargo test -p protocole
```

## Configuration par variables d'environnement

```rust
// src/config.rs
use std::env;
use thiserror::Error;

#[derive(Error, Debug)]
pub enum ErreurConfig {
    #[error("Variable d'environnement manquante : {0}")]
    VarManquante(String),
    #[error("Valeur invalide pour {variable} : {valeur} ({raison})")]
    ValeurInvalide { variable: String, valeur: String, raison: String },
}

#[derive(Debug, Clone)]
pub struct Config {
    pub hôte: String,
    pub port: u16,
    pub url_base_de_données: String,
    pub secret_jwt: String,
    pub niveau_log: String,
    pub taille_pool_connexions: u32,
}

impl Config {
    pub fn depuis_env() -> Result<Self, ErreurConfig> {
        Ok(Config {
            hôte: env_ou_défaut("HOST", "0.0.0.0"),
            port: env_parsé("PORT", 8080)?,
            url_base_de_données: env_requis("DATABASE_URL")?,
            secret_jwt: env_requis("JWT_SECRET")?,
            niveau_log: env_ou_défaut("LOG_LEVEL", "info"),
            taille_pool_connexions: env_parsé("DB_POOL_SIZE", 10)?,
        })
    }
}

fn env_requis(clé: &str) -> Result<String, ErreurConfig> {
    env::var(clé).map_err(|_| ErreurConfig::VarManquante(clé.to_string()))
}

fn env_ou_défaut(clé: &str, défaut: &str) -> String {
    env::var(clé).unwrap_or_else(|_| défaut.to_string())
}

fn env_parsé<T: std::str::FromStr>(clé: &str, défaut: T) -> Result<T, ErreurConfig>
where T::Err: std::fmt::Display
{
    match env::var(clé) {
        Ok(val) => val.parse::<T>().map_err(|e| ErreurConfig::ValeurInvalide {
            variable: clé.to_string(),
            valeur: val,
            raison: e.to_string(),
        }),
        Err(_) => Ok(défaut),
    }
}
```

## Logging structuré avec tracing

```toml
[dependencies]
tracing = "0.1"
tracing-subscriber = { version = "0.3", features = ["env-filter", "json"] }
```

```rust
// src/main.rs
use tracing::{info, warn, error, instrument};
use tracing_subscriber::{layer::SubscriberExt, util::SubscriberInitExt};

fn initialiser_tracing(niveau: &str) {
    tracing_subscriber::registry()
        .with(
            tracing_subscriber::EnvFilter::try_from_default_env()
                .unwrap_or_else(|_| niveau.into())
        )
        .with(
            // En développement : logs colorés lisibles
            // En production : JSON structuré pour les agrégateurs (Datadog, ELK)
            if std::env::var("RUST_LOG_JSON").is_ok() {
                tracing_subscriber::fmt::layer().json().boxed()
            } else {
                tracing_subscriber::fmt::layer().boxed()
            }
        )
        .init();
}

// Instrumenter une fonction — tous les appels seront loggés
#[instrument(skip(db), fields(utilisateur_id = %id))]
async fn obtenir_utilisateur(id: u64, db: &Database) -> Result<Utilisateur, Erreur> {
    info!("Récupération de l'utilisateur");
    
    let user = db.trouver_par_id(id).await?;
    
    if user.est_suspendu() {
        warn!(raison = %user.raison_suspension, "Accès à un compte suspendu");
    }
    
    info!(nom = %user.nom, "Utilisateur trouvé");
    Ok(user)
}

#[tokio::main]
async fn main() -> anyhow::Result<()> {
    let config = Config::depuis_env()?;
    initialiser_tracing(&config.niveau_log);
    
    info!(
        hôte = %config.hôte,
        port = config.port,
        "Démarrage du serveur"
    );
    
    // ...
    Ok(())
}
```

Sortie JSON en production :
```json
{"timestamp":"2024-01-15T10:23:45Z","level":"INFO","target":"mon_service","message":"Démarrage du serveur","hôte":"0.0.0.0","port":8080}
{"timestamp":"2024-01-15T10:23:45.123Z","level":"INFO","target":"mon_service::handlers","message":"Récupération de l'utilisateur","utilisateur_id":"42"}
```

## Makefile / justfile — Automatiser les tâches courantes

```makefile
# Makefile
.PHONY: dev test lint fmt build release docker

dev:
	cargo watch -x run

test:
	cargo test --workspace

lint:
	cargo clippy --workspace -- -D warnings

fmt:
	cargo fmt --all

build:
	cargo build --workspace

release:
	cargo build --workspace --release

check: fmt lint test
	@echo "✅ Tout est OK"

docker:
	docker build -t mon-service:latest .
```

## .cargo/config.toml — Configuration locale

```toml
# .cargo/config.toml
[build]
# Utiliser mold (linker rapide) en développement
# linker = "clang"
# rustflags = ["-C", "link-arg=-fuse-ld=mold"]

[env]
# Variables d'environnement par défaut pour le développement
DATABASE_URL = "postgresql://localhost/monapp_dev"
RUST_LOG = "debug"

[alias]
# Raccourcis personnalisés
t = "test --workspace"
b = "build --workspace"
r = "run"
c = "clippy --workspace -- -D warnings"
```

## Résumé — Structure idéale

```
Projet simple         → src/ avec modules
Projet multi-services → Workspace avec crates/
Configuration         → Variables d'environnement + struct Config
Logging               → tracing + JSON en production
Tâches               → Makefile ou justfile
```
