## SQLx — La bibliothèque SQL vérifiée à la compilation

SQLx est unique : elle vérifie vos requêtes SQL à la **compilation** contre une vraie base de données. Si votre SQL est invalide, votre code ne compile pas.

```toml
[dependencies]
sqlx = { version = "0.7", features = [
    "runtime-tokio-rustls",
    "postgres",
    "uuid",
    "chrono",
    "json",
] }
```

```bash
# Installer la CLI SQLx
cargo install sqlx-cli --no-default-features --features postgres

# Créer et appliquer les migrations
sqlx migrate add init
sqlx migrate run

# Préparer les requêtes pour la vérification offline
cargo sqlx prepare
```

## Connexion et Pool

```rust
use sqlx::{PgPool, postgres::PgPoolOptions};

async fn créer_pool(url: &str) -> Result<PgPool, sqlx::Error> {
    PgPoolOptions::new()
        .max_connections(20)          // Connexions simultanées max
        .min_connections(2)           // Maintenir au minimum 2 connexions chaudes
        .acquire_timeout(std::time::Duration::from_secs(5))
        .idle_timeout(std::time::Duration::from_secs(600))
        .max_lifetime(std::time::Duration::from_secs(1800))
        .connect(url)
        .await
}
```

## Requêtes typées avec query_as!

```rust
use sqlx::FromRow;
use uuid::Uuid;
use chrono::{DateTime, Utc};

#[derive(Debug, FromRow, serde::Serialize)]
pub struct Produit {
    pub id: Uuid,
    pub nom: String,
    pub prix: sqlx::types::Decimal,
    pub stock: i32,
    pub catégorie: String,
    pub créé_le: DateTime<Utc>,
}

// La macro query_as! vérifie le SQL à la compilation
pub async fn obtenir_produit(pool: &PgPool, id: Uuid) -> sqlx::Result<Option<Produit>> {
    sqlx::query_as!(
        Produit,
        "SELECT id, nom, prix, stock, catégorie, créé_le FROM produits WHERE id = $1",
        id
    )
    .fetch_optional(pool)
    .await
}

pub async fn lister_par_catégorie(
    pool: &PgPool,
    catégorie: &str,
    limite: i64,
    offset: i64,
) -> sqlx::Result<Vec<Produit>> {
    sqlx::query_as!(
        Produit,
        r#"
        SELECT id, nom, prix, stock, catégorie, créé_le
        FROM produits
        WHERE catégorie = $1 AND stock > 0
        ORDER BY prix ASC
        LIMIT $2 OFFSET $3
        "#,
        catégorie, limite, offset
    )
    .fetch_all(pool)
    .await
}
```

## Transactions — Atomicité garantie

Une transaction garantit que toutes les opérations réussissent ou qu'aucune n'est appliquée.

```rust
use sqlx::{PgPool, Postgres, Transaction};

// Transfert de stock entre deux entrepôts
pub async fn transférer_stock(
    pool: &PgPool,
    produit_id: Uuid,
    source_id: Uuid,
    destination_id: Uuid,
    quantité: i32,
) -> Result<(), ErreurStockage> {
    // Démarrer une transaction
    let mut tx = pool.begin().await?;
    
    // Opération 1 : décrémenter la source
    let lignes = sqlx::query!(
        r#"
        UPDATE stocks
        SET quantité = quantité - $1
        WHERE produit_id = $2 AND entrepôt_id = $3 AND quantité >= $1
        "#,
        quantité, produit_id, source_id
    )
    .execute(&mut *tx)
    .await?
    .rows_affected();
    
    if lignes == 0 {
        // Stock insuffisant — ROLLBACK automatique quand tx est droppé
        return Err(ErreurStockage::StockInsuffisant);
    }
    
    // Opération 2 : incrémenter la destination
    sqlx::query!(
        r#"
        INSERT INTO stocks (produit_id, entrepôt_id, quantité)
        VALUES ($1, $2, $3)
        ON CONFLICT (produit_id, entrepôt_id)
        DO UPDATE SET quantité = stocks.quantité + $3
        "#,
        produit_id, destination_id, quantité
    )
    .execute(&mut *tx)
    .await?;
    
    // COMMIT — applique toutes les modifications
    tx.commit().await?;
    
    Ok(())
}
```

## Niveaux d'isolation des transactions

```rust
use sqlx::postgres::PgTransactionManager;

// Read Committed (défaut PostgreSQL)
let mut tx = pool.begin().await?;

// Repeatable Read — pour les rapports cohérents
let mut tx = pool
    .begin()
    .await?;
sqlx::query("SET TRANSACTION ISOLATION LEVEL REPEATABLE READ")
    .execute(&mut *tx)
    .await?;

// Serializable — protection totale contre les anomalies
sqlx::query("SET TRANSACTION ISOLATION LEVEL SERIALIZABLE")
    .execute(&mut *tx)
    .await?;
```

## Index et optimisation de requêtes

```sql
-- Migration d'optimisation
-- Index simple
CREATE INDEX CONCURRENTLY idx_produits_catégorie
    ON produits(catégorie);

-- Index composite (colonne la plus sélective en premier)
CREATE INDEX CONCURRENTLY idx_commandes_client_date
    ON commandes(client_id, date_commande DESC);

-- Index partiel (seulement les commandes actives)
CREATE INDEX CONCURRENTLY idx_commandes_actives
    ON commandes(client_id)
    WHERE statut IN ('en_attente', 'en_cours');

-- Index pour la recherche textuelle
CREATE INDEX idx_produits_recherche
    ON produits USING gin(to_tsvector('french', nom || ' ' || description));
```

```rust
// Requête avec recherche textuelle
pub async fn rechercher_produits(pool: &PgPool, terme: &str) -> sqlx::Result<Vec<Produit>> {
    sqlx::query_as!(
        Produit,
        r#"
        SELECT id, nom, prix, stock, catégorie, créé_le,
               ts_rank(to_tsvector('french', nom || ' ' || description),
                       plainto_tsquery('french', $1)) AS rang
        FROM produits
        WHERE to_tsvector('french', nom || ' ' || description) @@
              plainto_tsquery('french', $1)
        ORDER BY rang DESC
        LIMIT 20
        "#,
        terme
    )
    .fetch_all(pool)
    .await
}
```

## EXPLAIN ANALYZE — Diagnostiquer les requêtes lentes

```rust
pub async fn analyser_requête(pool: &PgPool, requête: &str) -> sqlx::Result<String> {
    let plan: String = sqlx::query_scalar(&format!("EXPLAIN ANALYZE {}", requête))
        .fetch_one(pool)
        .await?;
    
    Ok(plan)
}

// En production : logger les requêtes lentes automatiquement
pub async fn avec_monitoring<T>(
    pool: &PgPool,
    nom: &str,
    opération: impl std::future::Future<Output = sqlx::Result<T>>,
) -> sqlx::Result<T> {
    let début = std::time::Instant::now();
    let résultat = opération.await;
    let durée = début.elapsed();
    
    if durée > std::time::Duration::from_millis(100) {
        tracing::warn!(
            requête = nom,
            durée_ms = durée.as_millis(),
            "Requête lente détectée"
        );
    }
    
    résultat
}
```

## 🔥 Failure First — Simuler une corruption de données

```rust
// Test de robustesse : que se passe-t-il si la transaction est interrompue ?
#[tokio::test]
async fn transaction_interrompue_rollback() {
    let pool = créer_pool_test().await;
    
    let compte = obtenir_solde(&pool, "compte-1").await.unwrap();
    
    // Simuler une interruption au milieu d'un transfert
    let résultat = async {
        let mut tx = pool.begin().await?;
        
        décrémenter_solde(&mut tx, "compte-1", 100).await?;
        
        // Simuler une panne : le code panique avant le commit
        panic!("Panne simulée !");
        
        // Cette ligne n'est jamais atteinte
        tx.commit().await?;
        Ok::<(), sqlx::Error>(())
    };
    
    // catch_unwind pour attraper le panic
    let _ = std::panic::catch_unwind(|| {
        // En pratique, le drop de tx déclenche le ROLLBACK automatique
    });
    
    // Vérifier que le solde est intact (ROLLBACK automatique)
    let solde_après = obtenir_solde(&pool, "compte-1").await.unwrap();
    assert_eq!(compte, solde_après, "Le ROLLBACK doit avoir annulé le débit");
}
```
