## Le théorème CAP expliqué clairement

En 2000, Eric Brewer a formulé un théorème fondamental : **tout système distribué ne peut garantir simultanément que deux des trois propriétés suivantes** :

```
         Cohérence (C)
            /\
           /  \
          /    \
         / CA   \
        /   ou   \
       /  CP  AP  \
      ──────────────
  Disponibilité (A)  Tolérance aux partitions (P)
```

**C — Cohérence (Consistency) :**
Chaque lecture retourne la valeur la plus récente écrite, ou une erreur. Tous les nœuds voient les mêmes données au même moment.

**A — Disponibilité (Availability) :**
Chaque requête reçoit une réponse (pas une erreur), mais sans garantie d'être la plus récente.

**P — Tolérance aux partitions (Partition tolerance) :**
Le système continue de fonctionner même si des messages réseau entre nœuds sont perdus ou retardés.

> 💡 **La réalité :** Dans un réseau réel, les partitions réseau *arrivent*. Vous ne pouvez pas choisir "P = non". Le vrai choix est : **CP ou AP** — quand il y a une partition, sacrifiez-vous la cohérence ou la disponibilité ?

---

## Cohérence forte vs éventuelle

```
Cohérence forte (CP) :
Nœud 1 : écrit "valeur=42" → propagé → Nœud 2 lit "42"  ✅
Mais si la propagation échoue → refus de la lecture ou de l'écriture

Cohérence éventuelle (AP) :
Nœud 1 : écrit "valeur=42" → Nœud 2 lit peut-être encore "41" (temporairement) ⚠️
Mais le service reste disponible, et "éventuellement" tous les nœuds convergent vers 42
```

### Exemples réels

| Système | Modèle | Raisonnement |
|---|---|---|
| PostgreSQL (single node) | C | Pas de partition — tout sur un nœud |
| etcd / ZooKeeper | CP | Leader élu, cohérence forte, peut refuser si pas de quorum |
| DynamoDB | AP (configurable) | Disponibilité prioritaire, cohérence éventuelle par défaut |
| Cassandra | AP (tunable) | Choix par requête du niveau de cohérence |
| Redis Cluster | AP | Disponibilité, risque de perte de données en cas de partition |

## Réplication — Comment partager les données

### Réplication synchrone (forte cohérence)

```
Client → Nœud Primaire → [attendre ACK de tous les répliques] → Réponse OK
```

```rust
pub async fn écriture_synchrone(
    primaire: &Nœud,
    répliques: &[Nœud],
    clé: &str,
    valeur: &[u8],
) -> Result<(), ErreurDistribuée> {
    // Écrire sur le primaire
    primaire.écrire(clé, valeur).await?;
    
    // Attendre l'ACK de TOUS les répliques
    let mut confirmations = futures::future::join_all(
        répliques.iter().map(|r| r.répliquer(clé, valeur))
    ).await;
    
    // Si un seul réplique échoue → rollback
    if confirmations.iter().any(|r| r.is_err()) {
        primaire.supprimer(clé).await?;
        return Err(ErreurDistribuée::ÉchecRéplication);
    }
    
    Ok(())
}
```

**Pro :** Zéro perte de données en cas de panne
**Con :** Latence = latence du réplique le plus lent

### Réplication asynchrone (disponibilité)

```rust
pub async fn écriture_asynchrone(
    primaire: &Nœud,
    répliques: &[Nœud],
    clé: &str,
    valeur: &[u8],
) -> Result<(), ErreurDistribuée> {
    // Écrire sur le primaire → réponse immédiate au client
    primaire.écrire(clé, valeur).await?;
    
    // Répliquer en arrière-plan (fire and forget)
    for réplique in répliques {
        let r = réplique.clone();
        let k = clé.to_string();
        let v = valeur.to_vec();
        tokio::spawn(async move {
            if let Err(e) = r.répliquer(&k, &v).await {
                tracing::warn!("Réplication échouée : {}", e);
                // Ajouter à une file de retry...
            }
        });
    }
    
    Ok(())
}
```

### Réplication par quorum

Le meilleur compromis : écrire sur W nœuds, lire depuis R nœuds, avec W + R > N (N = total de nœuds).

```rust
pub struct ConfigQuorum {
    pub n: usize, // Nombre total de nœuds
    pub w: usize, // Nœuds d'écriture requis
    pub r: usize, // Nœuds de lecture requis
}

impl ConfigQuorum {
    /// Cohérence forte : W + R > N
    pub fn cohérence_forte(n: usize) -> Self {
        let quorum = n / 2 + 1;
        ConfigQuorum { n, w: quorum, r: quorum }
    }
    
    /// Haute disponibilité : W=1, R=1
    pub fn haute_disponibilité(n: usize) -> Self {
        ConfigQuorum { n, w: 1, r: 1 }
    }
}

pub async fn écriture_quorum(
    nœuds: &[Nœud],
    config: &ConfigQuorum,
    clé: &str,
    valeur: &[u8],
    version: u64,
) -> Result<(), ErreurDistribuée> {
    use futures::stream::{FuturesUnordered, StreamExt};
    
    let mut futures_écriture: FuturesUnordered<_> = nœuds.iter()
        .map(|n| n.écrire_avec_version(clé, valeur, version))
        .collect();
    
    let mut succès = 0;
    
    while let Some(résultat) = futures_écriture.next().await {
        if résultat.is_ok() {
            succès += 1;
            if succès >= config.w {
                return Ok(()); // Quorum atteint !
            }
        }
    }
    
    Err(ErreurDistribuée::QuorumNonAtteint { requis: config.w, obtenus: succès })
}
```

## CRDT — Cohérence sans coordination

Les CRDTs (Conflict-free Replicated Data Types) sont des structures de données qui peuvent être modifiées simultanément sur plusieurs nœuds sans coordination, et qui convergent toujours vers le même état.

```rust
/// G-Counter : compteur croissant seulement, sans coordination
#[derive(Debug, Clone)]
pub struct GCounter {
    nœud_id: String,
    compteurs: HashMap<String, u64>,
}

impl GCounter {
    pub fn nouveau(nœud_id: &str) -> Self {
        GCounter {
            nœud_id: nœud_id.to_string(),
            compteurs: HashMap::new(),
        }
    }
    
    pub fn incrémenter(&mut self) {
        *self.compteurs.entry(self.nœud_id.clone()).or_insert(0) += 1;
    }
    
    pub fn valeur(&self) -> u64 {
        self.compteurs.values().sum()
    }
    
    /// Fusionner avec un autre GCounter (opération commutative + associative)
    pub fn fusionner(&mut self, autre: &GCounter) {
        for (nœud, &valeur) in &autre.compteurs {
            let entrée = self.compteurs.entry(nœud.clone()).or_insert(0);
            *entrée = (*entrée).max(valeur);
        }
    }
}

#[test]
fn crdt_convergence() {
    let mut compteur_a = GCounter::nouveau("nœud-A");
    let mut compteur_b = GCounter::nouveau("nœud-B");
    
    // Modifications indépendantes (partition simulée)
    compteur_a.incrémenter();
    compteur_a.incrémenter();
    
    compteur_b.incrémenter();
    
    // Fusion — converge vers le même état quelle que soit l'ordre
    compteur_a.fusionner(&compteur_b);
    compteur_b.fusionner(&compteur_a);
    
    assert_eq!(compteur_a.valeur(), 3); // 2 + 1
    assert_eq!(compteur_b.valeur(), 3); // Convergence garantie
}
```
