## TCP — La fondation de toute communication réseau

TCP (Transmission Control Protocol) garantit trois propriétés fondamentales :
- **Fiabilité** : tous les paquets arriveront, ou une erreur sera signalée
- **Ordre** : les données arrivent dans l'ordre d'envoi
- **Contrôle de flux** : pas d'inondation du destinataire

Ces garanties ont un coût : la latence du handshake (SYN → SYN-ACK → ACK) et l'overhead de retransmission.

### TCP bas niveau avec Tokio

```rust
use tokio::net::{TcpListener, TcpStream};
use tokio::io::{AsyncReadExt, AsyncWriteExt};

// Serveur TCP écho
#[tokio::main]
async fn main() -> anyhow::Result<()> {
    let listener = TcpListener::bind("0.0.0.0:8000").await?;
    println!("Serveur TCP sur :8000");
    
    loop {
        let (socket, addr) = listener.accept().await?;
        println!("Connexion depuis {}", addr);
        
        tokio::spawn(async move {
            gérer_client(socket).await;
        });
    }
}

async fn gérer_client(mut socket: TcpStream) {
    let mut buffer = vec![0u8; 4096];
    
    loop {
        match socket.read(&mut buffer).await {
            Ok(0) => {
                println!("Client déconnecté");
                break;
            }
            Ok(n) => {
                // Écho : renvoyer ce qui a été reçu
                if socket.write_all(&buffer[..n]).await.is_err() {
                    break;
                }
            }
            Err(e) => {
                eprintln!("Erreur lecture : {}", e);
                break;
            }
        }
    }
}
```

### Protocole de longueur-préfixée (framing)

TCP est un flux d'octets — il ne connaît pas les "messages". Il faut un protocole de cadrage :

```rust
use tokio::io::{AsyncReadExt, AsyncWriteExt};
use tokio::net::TcpStream;

// Protocole : [4 octets longueur][données]
async fn envoyer_message(stream: &mut TcpStream, données: &[u8]) -> anyhow::Result<()> {
    let longueur = données.len() as u32;
    stream.write_u32(longueur).await?; // Big-endian
    stream.write_all(données).await?;
    Ok(())
}

async fn recevoir_message(stream: &mut TcpStream) -> anyhow::Result<Vec<u8>> {
    let longueur = stream.read_u32().await? as usize;
    
    if longueur > 10 * 1024 * 1024 { // Limite à 10 Mo
        anyhow::bail!("Message trop grand : {} octets", longueur);
    }
    
    let mut données = vec![0u8; longueur];
    stream.read_exact(&mut données).await?;
    Ok(données)
}
```

## HTTP/1.1 vs HTTP/2 vs HTTP/3

```
HTTP/1.1 :
Client → [req1] → Serveur → [rep1] → Client
Client → [req2] → Serveur → [rep2] → Client  (séquentiel)

HTTP/2 :
Client → [req1][req2][req3] → Serveur → [rep2][rep1][rep3]  (multiplexé)
         Une seule connexion TCP, streams entrelacés

HTTP/3 :
Client ← UDP/QUIC → Serveur  (pas de head-of-line blocking, meilleure reprise après perte)
```

## REST — Conception d'API robuste

```rust
// Conventions REST à respecter
//
// GET    /utilisateurs          → liste
// POST   /utilisateurs          → créer
// GET    /utilisateurs/{id}     → lire
// PUT    /utilisateurs/{id}     → remplacer complètement
// PATCH  /utilisateurs/{id}     → mise à jour partielle
// DELETE /utilisateurs/{id}     → supprimer
//
// Codes de statut HTTP :
// 200 OK              → succès lecture/mise à jour
// 201 Created         → création réussie (avec Location header)
// 204 No Content      → succès sans corps (DELETE)
// 400 Bad Request     → données invalides (validation)
// 401 Unauthorized    → non authentifié
// 403 Forbidden       → authentifié mais non autorisé
// 404 Not Found       → ressource inexistante
// 409 Conflict        → conflit (ex: email dupliqué)
// 422 Unprocessable   → données sémantiquement invalides
// 429 Too Many Req.   → rate limiting
// 500 Internal Error  → bug serveur (ne jamais exposer les détails)
```

## gRPC — API binaire haute performance

gRPC utilise Protocol Buffers pour la sérialisation et HTTP/2 pour le transport. Idéal pour les communications inter-services.

```toml
[dependencies]
tonic = "0.11"
prost = "0.12"

[build-dependencies]
tonic-build = "0.11"
```

```protobuf
// proto/utilisateurs.proto
syntax = "proto3";
package utilisateurs;

service ServiceUtilisateurs {
    rpc Créer (RequêteCréer) returns (Utilisateur);
    rpc Obtenir (RequêteObtenir) returns (Utilisateur);
    rpc Lister (RequêteLister) returns (stream Utilisateur);
}

message RequêteCréer {
    string nom = 1;
    string email = 2;
}

message RequêteObtenir {
    string id = 1;
}

message RequêteLister {
    int32 page = 1;
    int32 par_page = 2;
}

message Utilisateur {
    string id = 1;
    string nom = 2;
    string email = 3;
    int64 créé_le = 4;
}
```

```rust
// build.rs
fn main() -> Result<(), Box<dyn std::error::Error>> {
    tonic_build::compile_protos("proto/utilisateurs.proto")?;
    Ok(())
}
```

```rust
// Implémentation du service gRPC
use tonic::{transport::Server, Request, Response, Status};

pub mod proto {
    tonic::include_proto!("utilisateurs");
}

use proto::service_utilisateurs_server::{ServiceUtilisateurs, ServiceUtilisateursServer};
use proto::{Utilisateur, RequêteCréer, RequêteObtenir, RequêteLister};

#[derive(Default)]
struct MonService;

#[tonic::async_trait]
impl ServiceUtilisateurs for MonService {
    async fn créer(
        &self,
        request: Request<RequêteCréer>,
    ) -> Result<Response<Utilisateur>, Status> {
        let req = request.into_inner();
        
        // Validation
        if req.nom.is_empty() {
            return Err(Status::invalid_argument("Le nom est requis"));
        }
        
        let user = Utilisateur {
            id: uuid::Uuid::new_v4().to_string(),
            nom: req.nom,
            email: req.email,
            créé_le: chrono::Utc::now().timestamp(),
        };
        
        Ok(Response::new(user))
    }
    
    async fn obtenir(
        &self,
        request: Request<RequêteObtenir>,
    ) -> Result<Response<Utilisateur>, Status> {
        // Simulation
        Ok(Response::new(Utilisateur {
            id: request.into_inner().id,
            nom: "Alice".to_string(),
            email: "alice@example.com".to_string(),
            créé_le: 0,
        }))
    }
    
    type ListerStream = tokio_stream::wrappers::ReceiverStream<Result<Utilisateur, Status>>;
    
    async fn lister(
        &self,
        _request: Request<RequêteLister>,
    ) -> Result<Response<Self::ListerStream>, Status> {
        let (tx, rx) = tokio::sync::mpsc::channel(16);
        
        // Streaming : envoyer des résultats au fur et à mesure
        tokio::spawn(async move {
            for i in 0..10 {
                let user = Utilisateur {
                    id: uuid::Uuid::new_v4().to_string(),
                    nom: format!("Utilisateur {}", i),
                    email: format!("user{}@example.com", i),
                    créé_le: 0,
                };
                if tx.send(Ok(user)).await.is_err() {
                    break;
                }
            }
        });
        
        Ok(Response::new(tokio_stream::wrappers::ReceiverStream::new(rx)))
    }
}

#[tokio::main]
async fn main() -> anyhow::Result<()> {
    let addr = "[::1]:50051".parse()?;
    
    Server::builder()
        .add_service(ServiceUtilisateursServer::new(MonService))
        .serve(addr)
        .await?;
    
    Ok(())
}
```

## Sérialisation : JSON vs Protocol Buffers vs MessagePack

```
Format        | Taille  | Vitesse | Lisible | Schéma
─────────────────────────────────────────────────────
JSON          | 100%    | moyen   | oui     | optionnel
MessagePack   | ~50%    | rapide  | non     | non
Protocol Buf  | ~30-40% | très rapide | non | oui (strict)
Bincode       | ~25%    | très rapide | non | Rust-only
```

```rust
// Comparer JSON vs Bincode vs serde_json
use serde::{Serialize, Deserialize};

#[derive(Serialize, Deserialize)]
struct Événement {
    id: u64,
    type_: String,
    données: Vec<u8>,
    horodatage: i64,
}

fn comparer_formats(événement: &Événement) {
    // JSON
    let json = serde_json::to_string(événement).unwrap();
    println!("JSON : {} octets", json.len());
    
    // MessagePack
    let msgpack = rmp_serde::to_vec(événement).unwrap();
    println!("MessagePack : {} octets", msgpack.len());
    
    // Bincode (le plus compact pour Rust)
    let bincode = bincode::serialize(événement).unwrap();
    println!("Bincode : {} octets", bincode.len());
}
```

## Timeout et retry — Résilience réseau

```rust
use tokio::time::{timeout, Duration};
use std::time::Duration as StdDuration;

async fn appel_http_résilient(url: &str) -> anyhow::Result<String> {
    let client = reqwest::Client::builder()
        .timeout(StdDuration::from_secs(5))
        .build()?;
    
    let max_tentatives = 3;
    let mut délai = StdDuration::from_millis(100);
    
    for tentative in 1..=max_tentatives {
        match client.get(url).send().await {
            Ok(rep) if rep.status().is_success() => {
                return Ok(rep.text().await?);
            }
            Ok(rep) if rep.status().is_server_error() => {
                // 5xx → retry
                tracing::warn!(
                    tentative,
                    statut = rep.status().as_u16(),
                    "Erreur serveur, retry dans {:?}", délai
                );
            }
            Ok(rep) => {
                // 4xx → ne pas retry (erreur client)
                anyhow::bail!("Erreur client HTTP {}", rep.status());
            }
            Err(e) if tentative < max_tentatives => {
                tracing::warn!(tentative, "Erreur réseau : {}, retry...", e);
            }
            Err(e) => return Err(e.into()),
        }
        
        tokio::time::sleep(délai).await;
        délai *= 2; // Backoff exponentiel
    }
    
    anyhow::bail!("Échec après {} tentatives", max_tentatives)
}
```
