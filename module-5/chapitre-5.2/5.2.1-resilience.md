## Pourquoi la résilience doit être conçue, pas espérée

Un système distribué qui fonctionne bien en conditions normales n'est pas nécessairement résilient. La vraie question est : que se passe-t-il quand un service dépendant répond en 10 secondes ? Quand la base de données est saturée ? Quand un nœud disparaît ?

Sans mécanismes de résilience, une panne partielle peut se propager en cascade et faire tomber l'ensemble du système — même les parties qui fonctionnaient parfaitement.

## Circuit Breaker — Couper avant de s'effondrer

Le Circuit Breaker (coupe-circuit) est emprunté à l'électricité : quand le courant dépasse un seuil dangereux, on coupe le circuit pour protéger le reste.

```
États du Circuit Breaker :

FERMÉ (normal) → requêtes passent → succès comptabilisés
    │ trop d'échecs
    ▼
OUVERT (protection) → requêtes bloquées immédiatement → erreur rapide
    │ délai de récupération écoulé
    ▼
SEMI-OUVERT (test) → quelques requêtes passent → succès ? → FERMÉ / OUVERT
```

```rust
// src/circuit_breaker.rs
use std::sync::Arc;
use std::time::{Duration, Instant};
use tokio::sync::Mutex;

#[derive(Debug, Clone, PartialEq)]
pub enum ÉtatCB { Fermé, Ouvert, SemiOuvert }

#[derive(Debug)]
struct ÉtatInterne {
    état: ÉtatCB,
    compteur_échecs: u32,
    dernier_échec: Option<Instant>,
    compteur_succès_semi: u32,
}

#[derive(Clone)]
pub struct CircuitBreaker {
    nom: String,
    config: ConfigCB,
    état: Arc<Mutex<ÉtatInterne>>,
}

#[derive(Clone)]
pub struct ConfigCB {
    pub seuil_échecs: u32,       // Nombre d'échecs avant ouverture
    pub délai_récupération: Duration, // Temps avant de passer en semi-ouvert
    pub seuil_succès_semi: u32,  // Succès requis en semi-ouvert pour fermer
}

impl Default for ConfigCB {
    fn default() -> Self {
        ConfigCB {
            seuil_échecs: 5,
            délai_récupération: Duration::from_secs(30),
            seuil_succès_semi: 2,
        }
    }
}

#[derive(Debug, thiserror::Error)]
pub enum ErreurCB {
    #[error("Circuit ouvert — service {0} indisponible")]
    CircuitOuvert(String),
    #[error("Erreur de l'opération : {0}")]
    OpérationÉchouée(String),
}

impl CircuitBreaker {
    pub fn nouveau(nom: impl Into<String>, config: ConfigCB) -> Self {
        CircuitBreaker {
            nom: nom.into(),
            config,
            état: Arc::new(Mutex::new(ÉtatInterne {
                état: ÉtatCB::Fermé,
                compteur_échecs: 0,
                dernier_échec: None,
                compteur_succès_semi: 0,
            })),
        }
    }

    pub async fn exécuter<T, E, F, Fut>(&self, opération: F) -> Result<T, ErreurCB>
    where
        F: FnOnce() -> Fut,
        Fut: std::future::Future<Output = Result<T, E>>,
        E: std::fmt::Display,
    {
        // Vérifier si la requête peut passer
        {
            let mut état = self.état.lock().await;
            match état.état {
                ÉtatCB::Ouvert => {
                    // Vérifier si le délai est écoulé
                    let peut_tenter = état.dernier_échec
                        .map(|t| t.elapsed() >= self.config.délai_récupération)
                        .unwrap_or(false);

                    if peut_tenter {
                        état.état = ÉtatCB::SemiOuvert;
                        état.compteur_succès_semi = 0;
                        tracing::info!(cb = %self.nom, "Circuit → Semi-Ouvert (test)");
                    } else {
                        return Err(ErreurCB::CircuitOuvert(self.nom.clone()));
                    }
                }
                ÉtatCB::Fermé | ÉtatCB::SemiOuvert => {}
            }
        }

        // Exécuter l'opération
        match opération().await {
            Ok(résultat) => {
                self.enregistrer_succès().await;
                Ok(résultat)
            }
            Err(e) => {
                self.enregistrer_échec().await;
                Err(ErreurCB::OpérationÉchouée(e.to_string()))
            }
        }
    }

    async fn enregistrer_succès(&self) {
        let mut état = self.état.lock().await;
        match état.état {
            ÉtatCB::SemiOuvert => {
                état.compteur_succès_semi += 1;
                if état.compteur_succès_semi >= self.config.seuil_succès_semi {
                    état.état = ÉtatCB::Fermé;
                    état.compteur_échecs = 0;
                    tracing::info!(cb = %self.nom, "Circuit → Fermé (rétabli)");
                }
            }
            ÉtatCB::Fermé => {
                état.compteur_échecs = état.compteur_échecs.saturating_sub(1);
            }
            ÉtatCB::Ouvert => {}
        }
    }

    async fn enregistrer_échec(&self) {
        let mut état = self.état.lock().await;
        état.compteur_échecs += 1;
        état.dernier_échec = Some(Instant::now());

        if état.compteur_échecs >= self.config.seuil_échecs
            || état.état == ÉtatCB::SemiOuvert
        {
            état.état = ÉtatCB::Ouvert;
            tracing::warn!(
                cb = %self.nom,
                échecs = état.compteur_échecs,
                "⚡ Circuit OUVERT"
            );
        }
    }

    pub async fn état(&self) -> ÉtatCB {
        self.état.lock().await.état.clone()
    }
}

// Usage
#[tokio::main]
async fn main() {
    let cb = CircuitBreaker::nouveau(
        "service-paiement",
        ConfigCB {
            seuil_échecs: 3,
            délai_récupération: Duration::from_secs(10),
            seuil_succès_semi: 1,
        },
    );

    for _ in 0..10 {
        let résultat = cb.exécuter(|| async {
            appel_service_externe().await
        }).await;

        match résultat {
            Ok(val) => println!("Succès : {}", val),
            Err(ErreurCB::CircuitOuvert(_)) => {
                println!("⚡ Circuit ouvert — réponse immédiate sans attendre");
            }
            Err(ErreurCB::OpérationÉchouée(e)) => {
                println!("Échec : {}", e);
            }
        }

        tokio::time::sleep(Duration::from_millis(500)).await;
    }
}

async fn appel_service_externe() -> Result<String, String> {
    // Simuler des pannes
    static COMPTEUR: std::sync::atomic::AtomicU32 = std::sync::atomic::AtomicU32::new(0);
    let n = COMPTEUR.fetch_add(1, std::sync::atomic::Ordering::Relaxed);
    if n % 4 != 0 { Err("Service temporairement indisponible".to_string()) }
    else { Ok("Paiement traité".to_string()) }
}
```

## Bulkhead — Isolation des ressources

Le pattern Bulkhead (cloison étanche) isole les ressources pour qu'une panne d'un composant n'affecte pas les autres.

```rust
use tokio::sync::Semaphore;
use std::sync::Arc;

pub struct Bulkhead {
    nom: String,
    sémaphore: Arc<Semaphore>,
    métriques_rejet: std::sync::atomic::AtomicU64,
}

impl Bulkhead {
    pub fn nouveau(nom: impl Into<String>, capacité: usize) -> Self {
        Bulkhead {
            nom: nom.into(),
            sémaphore: Arc::new(Semaphore::new(capacité)),
            métriques_rejet: std::sync::atomic::AtomicU64::new(0),
        }
    }

    pub async fn exécuter<T, F, Fut>(&self, opération: F) -> Result<T, &'static str>
    where
        F: FnOnce() -> Fut,
        Fut: std::future::Future<Output = T>,
    {
        // try_acquire : échoue immédiatement si plein (ne bloque pas)
        match self.sémaphore.clone().try_acquire_owned() {
            Ok(_permis) => Ok(opération().await),
            Err(_) => {
                self.métriques_rejet.fetch_add(1, std::sync::atomic::Ordering::Relaxed);
                tracing::warn!(bulkhead = %self.nom, "Requête rejetée — capacité maximale");
                Err("Service surchargé — réessayez plus tard")
            }
        }
    }

    pub fn rejets(&self) -> u64 {
        self.métriques_rejet.load(std::sync::atomic::Ordering::Relaxed)
    }
}

// Application : isoler les appels vers différents backends
struct ServiceOrchestration {
    bh_paiement: Bulkhead,   // Max 20 appels simultanés vers le service paiement
    bh_notifications: Bulkhead, // Max 50 appels vers les notifications
    bh_analytique: Bulkhead, // Max 5 (non-critique)
}
```

## Retry avec backoff exponentiel et jitter

```rust
use std::time::Duration;
use rand::Rng;

pub struct ConfigRetry {
    pub max_tentatives: u32,
    pub délai_initial: Duration,
    pub délai_max: Duration,
    pub multiplicateur: f64,
    pub jitter: bool, // Ajoute du bruit pour éviter les tempêtes synchronisées
}

impl Default for ConfigRetry {
    fn default() -> Self {
        ConfigRetry {
            max_tentatives: 3,
            délai_initial: Duration::from_millis(100),
            délai_max: Duration::from_secs(30),
            multiplicateur: 2.0,
            jitter: true,
        }
    }
}

pub async fn avec_retry<T, E, F, Fut>(
    config: &ConfigRetry,
    mut opération: F,
) -> Result<T, E>
where
    F: FnMut() -> Fut,
    Fut: std::future::Future<Output = Result<T, E>>,
    E: std::fmt::Debug,
{
    let mut délai = config.délai_initial;

    for tentative in 1..=config.max_tentatives {
        match opération().await {
            Ok(val) => return Ok(val),
            Err(e) if tentative == config.max_tentatives => {
                tracing::error!(tentative = tentative, "Toutes les tentatives épuisées : {:?}", e);
                return Err(e);
            }
            Err(e) => {
                let délai_réel = if config.jitter {
                    let jitter = rand::thread_rng().gen_range(0..=délai.as_millis() as u64 / 2);
                    délai + Duration::from_millis(jitter)
                } else {
                    délai
                };

                tracing::warn!(
                    tentative = tentative,
                    délai_ms = délai_réel.as_millis(),
                    "Tentative échouée : {:?}, retry dans {:?}", e, délai_réel
                );

                tokio::time::sleep(délai_réel).await;

                délai = (délai.mul_f64(config.multiplicateur)).min(config.délai_max);
            }
        }
    }
    unreachable!()
}
```

## Chaos Engineering — Tester la résilience

```rust
// src/chaos.rs — Middleware de chaos pour les tests
use axum::{extract::Request, middleware::Next, response::Response};
use rand::Rng;
use std::time::Duration;

#[derive(Clone)]
pub struct ConfigChaos {
    pub taux_latence: f64,       // Probabilité d'ajouter une latence
    pub latence_min_ms: u64,
    pub latence_max_ms: u64,
    pub taux_erreur: f64,        // Probabilité d'erreur 500
    pub taux_timeout: f64,       // Probabilité de timeout
    pub activé: bool,
}

pub async fn middleware_chaos(
    axum::extract::State(config): axum::extract::State<std::sync::Arc<ConfigChaos>>,
    req: Request,
    suivant: Next,
) -> Result<Response, axum::http::StatusCode> {
    if !config.activé { return Ok(suivant.run(req).await); }

    let mut rng = rand::thread_rng();

    // Injecter une latence
    if rng.gen_bool(config.taux_latence) {
        let délai = rng.gen_range(config.latence_min_ms..config.latence_max_ms);
        tracing::warn!("🔥 Chaos: latence +{}ms", délai);
        tokio::time::sleep(Duration::from_millis(délai)).await;
    }

    // Injecter une erreur
    if rng.gen_bool(config.taux_erreur) {
        tracing::warn!("🔥 Chaos: erreur 503 injectée");
        return Err(axum::http::StatusCode::SERVICE_UNAVAILABLE);
    }

    // Injecter un timeout (simule une réponse qui n'arrive jamais)
    if rng.gen_bool(config.taux_timeout) {
        tracing::warn!("🔥 Chaos: timeout infini injecté");
        tokio::time::sleep(Duration::from_secs(3600)).await;
        return Err(axum::http::StatusCode::GATEWAY_TIMEOUT);
    }

    Ok(suivant.run(req).await)
}
```

## Résumé — Checklist de résilience

| Pattern | Protège contre | Implémentation |
|---|---|---|
| Circuit Breaker | Cascades de pannes | `CircuitBreaker::exécuter()` |
| Retry + backoff | Pannes transitoires | `avec_retry()` |
| Timeout | Blocages indéfinis | `tokio::time::timeout()` |
| Bulkhead | Saturation par un composant | `Bulkhead::exécuter()` |
| Jitter | Tempêtes de retry synchronisées | `rand::gen_range()` dans le délai |
| Health checks | Détection de panne | `/santé` endpoint |
| Graceful degradation | Service dégradé mais fonctionnel | Fallback + cache |
